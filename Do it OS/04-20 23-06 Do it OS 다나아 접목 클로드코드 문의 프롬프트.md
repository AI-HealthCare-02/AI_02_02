# Do it OS를 다나아 웹서비스에 접목하기 위한 Claude Code 문의 프롬프트

아래 내용을 Claude Code에게 그대로 전달해서, 다나아 웹서비스에 `Do it OS`를 어떻게 안전하게 접목할지 검토받고 싶습니다.

---

## 요청 목적

현재 다나아 웹서비스는 AI 채팅을 중심으로 사용자가 자연스럽게 건강 정보를 남기고, 리포트와 챌린지를 통해 생활습관을 관리하는 서비스입니다.

여기에 `Do it OS`라는 새 확장 기능을 붙이고 싶습니다.

`Do it OS`는 단순 할 일 관리 앱이 아니라, 사용자가 AI 채팅을 하면서 자연스럽게 흘린 생각, 할 일, 일정, 프로젝트, 걱정, 건강 단서를 모아두고, 사용자가 확인한 것만 분류해서 저장하는 생활 관리 레이어입니다.

핵심 목적은 다나아가 다른 건강 서비스처럼 “건강 기록만 요구하는 서비스”가 아니라, 사용자가 일상 업무와 생각 정리를 하면서 자연스럽게 건강 단서도 함께 수집하고 관리하게 만드는 것입니다.

## Do it OS의 이론적 배경

이 기능은 아래 개념을 바탕으로 합니다.

1. `브레인덤프`
   - 머릿속에 있는 생각, 걱정, 할 일, 아이디어를 일단 밖으로 꺼내는 방식입니다.
   - 다나아에서는 `생각넣기` 또는 AI 채팅 중 나온 저장 후보가 여기에 해당합니다.

2. `자이가르닉 효과(Zeigarnik Effect)`
   - 끝나지 않은 일, 해결되지 않은 일, 해야 할 일이 머릿속에 계속 남아 반복해서 떠오르는 현상입니다.
   - 예: 자기 전에 내일 할 일이 계속 생각나서 잠이 잘 안 오는 상태.
   - Do it OS는 이런 미완료 생각을 밖으로 꺼내고, 위치를 정해주어 머릿속 부담을 줄이는 역할을 합니다.

3. `수면 전 인지 각성(pre-sleep cognitive arousal)`
   - 자기 전에도 머리가 계속 깨어 있고 생각이 멈추지 않는 상태입니다.
   - 할 일과 걱정을 밖으로 꺼내면 잠들기 전 생각 반복을 줄이는 데 도움을 줄 수 있습니다.

4. `GTD(Getting Things Done)`
   - 모든 생각을 수집하고, 실행 가능한지 판단하고, 다음 행동으로 바꾸는 방법론입니다.
   - 다나아에서는 `생각넣기 → 분류 확인 → 확인 후 저장` 흐름으로 재해석하고 싶습니다.

5. `PARA`
   - 정보를 목적에 따라 `Project / Area / Resource / Archive`로 나누는 방식입니다.
   - 다나아에서는 원본 PARA 전체를 그대로 복사하지 않고, `프로젝트 / 일정 / 노트 / 건강 단서` 정도로 가볍게 재구성하고 싶습니다.

6. `Second Brain`
   - 내 머리 밖에 만드는 두 번째 기억/정리 시스템입니다.
   - 다나아에서는 이것을 건강 서비스에 맞게 축소해서, 사용자의 일상 맥락과 건강 단서를 함께 보관하는 구조로 쓰고 싶습니다.

## 중요한 제품 방향

Do it OS는 다나아의 중심 UX를 바꾸면 안 됩니다.

반드시 유지할 것:

- 다나아의 기본은 여전히 `AI 채팅 중심`입니다.
- 기존 `AI 채팅 / 리포트 / 챌린지 / 설정` 구조를 깨지 않습니다.
- 오른쪽 Today 패널은 건강 입력 역할을 계속 유지합니다.
- Do it OS는 건강 기록을 대체하지 않고, 사용자의 생각과 생활 맥락을 정리하는 보조 기능입니다.
- AI가 자동으로 건강 기록을 저장하면 안 됩니다.
- AI는 저장 후보만 만들고, 사용자가 확인한 것만 저장해야 합니다.
- 의료 진단, 치료 권유, 약물 안내 같은 기능으로 확장하면 안 됩니다.

## 현재 생각 중인 적용 방향

현재는 `확장형` 방향이 가장 끌립니다.

원하는 구조:

- 왼쪽 사이드바에 `Do it OS` 메뉴를 추가합니다.
- `Do it OS`를 누르면 하위 목록이 펼쳐집니다.
- 하위 목록 예시:
  - 대시보드
  - 생각넣기
  - 분류 확인
  - 프로젝트
  - 일정
  - 노트
- `/app/do-it` 또는 `/app/do-it-os` 같은 별도 화면에서 본문 전체 대시보드를 보여줍니다.
- 이 화면에서는 오른쪽 사이드 패널 없이, 중앙 전체를 Do it OS 대시보드로 사용합니다.
- 대시보드에는 아래 요소가 한 페이지에 보여야 합니다.
  - 생각넣기
  - 분류 확인
  - 오늘 할 일
  - 일정 후보
  - 프로젝트
  - 노트
  - 건강 단서
  - 저장 원칙

## 구현 시 반드시 지켜야 할 제약

- 기존 다나아 기능을 깨지 말아야 합니다.
- 기존 DB schema를 바로 크게 바꾸지 말아야 합니다.
- 원본 Do it OS 또는 Life OS 프로젝트의 코드를 그대로 복사하지 말고, UX 개념만 다나아에 맞게 재설계해야 합니다.
- `codex_NEW_DESKTOP_Dev_phase4`는 참고 프로젝트입니다. 실제 이식 기준 코드는 다나아 저장소 구조에 맞춰 새로 설계해야 합니다.
- 자동 저장 금지.
- 사용자가 확인한 항목만 저장.
- 건강 기록과 건강 단서는 분리.
- 건강 단서는 진단이 아니라 생활 맥락 참고 정보로만 사용.
- 처음부터 큰 기능을 다 만들지 말고, MVP 단계로 나누어야 합니다.

## 원하는 결론 방향

저는 Do it OS를 다나아의 핵심 방향으로 확장하고 싶지만, 한 번에 대규모로 붙여서 기존 서비스를 망가뜨리고 싶지는 않습니다.

그래서 Claude Code는 바로 구현하지 말고, 먼저 현재 코드 구조를 확인한 뒤 안전한 단계별 접목 전략을 제안해주세요.

특히 “다나아의 AI 채팅 중심 구조를 유지하면서 Do it OS를 어디까지 붙이면 좋은지”를 현실적으로 판단해주세요.

검토 순서는 아래처럼 진행해주세요.

1. 다나아 웹서비스 현재 구조 확인
2. `codex_NEW_DESKTOP_Dev_phase4` 원본 참고 프로젝트 구조와 완성도 확인
3. Do it OS 목업 2개 확인
4. 원본 프로젝트에서 가져올 수 있는 UX 개념과 가져오면 안 되는 구현 구조 분리
5. 다나아에 맞는 MVP 범위 제안
6. 구현 전 필요한 DB/API/프론트 영향 분석
7. 최종적으로 “지금 바로 붙일 수 있는 것 / 나중에 해야 하는 것 / 하지 말아야 하는 것”으로 정리
