# 코드 품질 검증

## 검증 대상
변경 파일: !`git diff --name-only HEAD 2>/dev/null || echo "변경사항 없음"`
변경 통계: !`git diff --stat HEAD 2>/dev/null || echo "변경사항 없음"`

IMPORTANT: 이 검증은 3단계로 구성된다. 하나라도 FAIL이면 수정 후 해당 항목만 재검증.
IMPORTANT: "구조가 있는가?"만 보지 말고 **"숫자가 정확한가?"**를 반드시 확인할 것.

---

## 1단계: 자동 검증 (도구로 반드시 실행 — 가장 중요)

### 1-1. Grep 전체 검색 (숫자/명칭 정확성)
변경 작업에서 바뀐 모든 숫자, 이름, 개수를 Grep으로 **전체 프로젝트 검색**:
- 변경 **전** 값이 아직 남아있으면 FAIL (예: "8개 테이블"→"9개"로 바꿨는데 다른 파일에 "8개" 남아있음)
- 검색 범위: 수정 파일뿐 아니라 `docs/`, `tools/`, `.claude/rules/`, 루트 `.md` 파일 전부
- **반드시 Grep 도구를 사용하여 실행하고 결과를 보고할 것**

### 1-2. 직접 카운트 (개수 주장 검증)
문서나 코드에서 "N개 필드", "N개 테이블", "N개 출처" 등 **개수를 주장하는 곳**은:
- 코드에서 필드 리스트를 Read로 읽고 **하나씩 세서** 실제 개수 확인
- 문서의 숫자와 코드의 실제 개수가 일치하지 않으면 FAIL
- "대충 맞겠지"로 넘어가면 안 됨 — 반드시 카운트 결과를 숫자로 보고

### 1-3. 실행 결과 확인 (출력 검증)
실행 가능한 스크립트가 있으면 실행하여 결과로 검증:
- Python: `py_compile` 통과 확인
- 엑셀 스크립트: 시트 수, 시트 이름 순서, 각 시트 행 수 출력 확인
- Next.js: `npm run build` 에러 없음 확인

---

## 2단계: 코드 품질 검증

### 2-1. 기존 패턴 준수
- 네이밍 컨벤션, 에러 핸들링, import 구조가 기존 코드와 일관적인가?
- 새 유틸 함수를 만들었는데 기존에 동일한 게 이미 있지 않은가?

### 2-2. 의존성 안전
- 수정한 파일을 import하는 다른 파일에 영향은? (Grep으로 검색)
- 함수 시그니처 변경 시 호출부도 모두 수정되었는가?

### 2-3. 보안
- 하드코딩된 비밀번호/API 키가 없는가?
- SQL 인젝션 / XSS 가능성이 없는가?
- 로그에 민감정보(혈압·혈당·건강기록)가 남지 않는가?

### 2-4. 다나아 도메인 규칙
- 의료적 진단/처방 표현이 없는가?
- 건강 데이터 최소 수집 원칙을 준수하는가?

### 2-5. 코드 품질
- Python: `ruff check <변경파일>` 실행
- 함수가 50줄을 초과하지 않는가?

### 2-6. 설계 일관성
- design-context.md 규칙과 충돌 없는가?
- 구현은 V8-8 데모 기준인가? (V8-7 이하 구버전 패턴 사용 시 FAIL)

### 2-7. 기술 기준
- `.claude/rules/eval-criteria.md`의 기술 기준 항목을 확인 (API 설계, P95 latency, 배포 안정성 등)

---

## 3단계: 교차 검증 (크로스 체크)

### 3-1. 문서-코드 일치성
같은 정보를 여러 파일에서 말할 때 **전부 일치하는지** 확인:
- 엑셀 스크립트의 필드 수 = 문서의 "N개 필드" 설명
- 엑셀 스크립트의 시트 순서 = 문서의 시트 번호 설명
- 코드의 변수명 = 문서의 필드명

### 3-2. 크로스파일 정합성
관련 문서가 여러 개일 때 숫자/명칭이 모두 동기화되어 있는지:
- 예: `docs/다나아_DB모델_팀공유_프롬프트.md`의 숫자 = `매일수집질문_DB설계.md`의 숫자 = 엑셀 스크립트 실제 값

## 출력 형식

```
| 검증 항목 | 결과 | 상세 |
|-----------|------|------|
| 1-1 Grep 전체검색 | PASS/FAIL | 검색한 키워드와 결과 건수 |
| 1-2 직접 카운트 | PASS/FAIL | 주장한 숫자 vs 실제 숫자 |
| 1-3 실행 결과 | PASS/FAIL | 출력 결과 요약 |
| 2-1~2-7 코드품질 | PASS/WARN/FAIL | 이슈 있으면 상세 |
| 3-1~3-2 교차검증 | PASS/FAIL | 불일치 있으면 상세 |
```

FAIL이 1건이라도 있으면 수정 후 해당 항목만 재검증하고 최종 보고.
