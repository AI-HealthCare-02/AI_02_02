# CLAUDE.md — 다나아 프로젝트 실행 가이드

## 1. 프로젝트 개요

- **다나아 (DA-NA-A)**: 만성질환(당뇨·고혈압) 생활습관 AI 건강관리 웹 서비스
- 필수 3기능: `당뇨 위험도 예측 모델` / `건강 추적 대시보드` / `생활습관 챌린지`
- 스택: **FastAPI 백엔드** + **Next.js 14 프론트엔드** + Docker

## 4. 작업 실행 규칙

### 4-1. "실행해줘" 받기 전 (질문/논의 모드)
사용자가 "실행해줘", "해줘", "적용해줘" 등 명시적으로 지시하기 전까지:
- 조사(Glob/Grep/Read), 설명, 제안, 플랜 제시**만** 한다
- 코드 수정, 파일 생성, 스크립트 실행 등 **실제 작업은 절대 하지 않는다**
- 사용자가 질문/지적을 해도 "바로 고치겠습니다"가 아니라 **설명 후 지시 대기**

### 4-2. "실행해줘" 받은 후 (실행 모드)
아래 4단계를 **자동으로 연속 실행** (추가 지시 불필요):
1. **구현** — 코드 수정/파일 생성
2. **§5 자기검증 5단계** — git diff, Grep, 패턴, 도메인, ruff/build
3. **/evaluate** — 3단계 검증 (자동 → 품질 → 교차)
4. **결과 보고** — PASS/WARN/FAIL 보고 (커밋은 사용자 지시 시에만)

### 4-3. 구현 전 자가 점검 (하나라도 NO면 먼저 조사)
1. 기존 파일을 Glob/Grep으로 검색했는가? → YES/NO
2. 네이밍 컨벤션을 확인했는가? → YES/NO
3. 수정할 파일을 import하는 다른 파일이 있는지? → YES/NO
4. 재사용 가능한 유틸/컴포넌트가 있는지? → YES/NO

### 4-4. 파일 수정 규칙
- 큰 파일(50줄 이상): Edit(부분 수정)만 사용. Write(전체 교체) 금지
- `archive/`, `docs/planning/`: 어떤 이유로도 수정 금지

---

## 5. 구현 후 자기 검증 (§4-2의 2단계)

"실행해줘" 후 구현이 끝나면, 결과 보고 전에 아래 5단계를 수행한다 (§4-2 자동 체인의 일부):

- [ ] **변경 리뷰**: `git diff`로 변경 내용 재확인
- [ ] **의존성 확인**: Grep으로 수정 파일을 import하는 다른 파일 검색 → 깨진 곳 없는지
- [ ] **패턴 일관성**: 네이밍·에러처리·import 구조가 기존 코드와 일관적인지
- [ ] **도메인 규칙**: 의료적 표현 금지, 민감정보 노출, 면책조항 누락 없는지
- [ ] **실행 검증 + 보안**: `ruff check`(Python) / `npm run build`(Next.js) + 하드코딩 비밀번호·XSS·SQL injection 없는지

→ PASS/WARN/FAIL 판정. FAIL이면 수정 후 재검증.
→ 이후 자동으로 `/evaluate` 실행 (§4-2의 3단계).

## 6. 플랜 품질 기준

IMPORTANT: `/plan` 또는 구현 계획 수립 시, 아래 항목이 포함되어야 한다:

1. **변경 파일 목록**: 수정/생성/삭제할 파일 경로 명시
2. **기존 코드 조사 결과**: 관련 기존 함수/패턴을 확인한 결과 (재사용 가능 여부)
3. **영향 범위**: 변경이 다른 파일/기능에 미치는 영향
4. **위험 요소**: 깨질 수 있는 부분, 주의할 점
5. **검증 방법**: 구현 후 어떻게 테스트/확인할 것인지 (Grep 전체 검색 + 직접 카운트 + 실행 결과 확인 포함)
6. **정량 근거**: 기술 선택에 수치·논문·데이터 근거 (해당 시)

### 플랜 모드 작업 방식 (자동 적용)

플랜 모드(`/plan` 또는 Shift+Tab) 진입 시 아래 규칙 자동 적용:

**에이전트 팀 규칙 (숫자 기준):**
- 조사 단계: **3개 에이전트 병렬** (각각 다른 관점으로 탐색). 2개 이하 금지
- 구현 후 교차 검증: **별도 에이전트 1개**로 fresh-context 검증 (자기가 만든 걸 자기가 검증 안 함)
- 단순 작업 (1파일, 10줄 이하 수정): 에이전트 팀 불필요

**기타:**
- 숫자/명칭 Grep 검증: 변경된 모든 숫자·이름을 Grep으로 전체 검색하여 누락 확인
- 최신 데모 기준: V8-8만 참조. 구버전 패턴 사용 금지
