그룹에 따라 일부 질문을 건너뛰어서 사용자마다 체감 화면 수가 다름.

| 10-1 | 평균 수면시간 | 가로 4지 | `sleep_duration_bucket` | 전체 | 인슐린 저항성, 6h이하 → 위험 28%↑ |

## 1. 당뇨 위험도 예측 모델

예측 모델은 처음 점수를 계산하고, 이후 매일 쌓이는 데이터로 점수를 더 정확하게 업데이트하는 용도.

| 질문 | 수집 주기 | 왜 필요한가 |
|------|----------|-----------|
| 수면 시간·질 (5택 복합) | 매일 | 6시간 이하 → 위험 28% 증가 (Cappuccio 2010) |
| 운동 시간 (분) | 매일 | 주 150분 달성 여부 → 위험 58% 감소 (DPP 2002) |
| 채소/과일 섭취 여부 | 매일 | FINDRISC(핀란드 당뇨 위험 점수표) 필수 변수 |
| 체중 | 매일~주간 | BMI 업데이트 → kg 감소마다 16% 위험 감소 |
| 복약 확인 (고혈압약) | 매일 | FINDRISC 필수 변수 — 혈압약 복용 여부 |
| 혈압 | 주 1-2회 | KDRS(한국형 당뇨 위험 점수) 변수 |
| 공복혈당 / HbA1c | 3-6개월마다 | 검사 결과 입력 — 가장 강력한 예측 변수 |

## 2. 건강 추적 대시보드

대시보드는 시간 흐름에 따른 변화 그래프를 보여주는 기능. 데이터가 쌓일수록 트렌드(추세)를 볼 수 있음.

| 질문 | DB 필드명 | 수집 주기 | 대시보드에서 보여주는 것 |
|------|----------|----------|----------------------|
| 수면 시간·질 | `sleep_quality`, `sleep_duration_bucket` | 매일 | 일별/주간 평균 수면 트렌드 |
| 아침 식사 | `breakfast_status` | 매일 | 식사 규칙성 트렌드 |
| 점심 식사 | `lunch_status` | 매일 | 식사 규칙성 트렌드 |
| 저녁 식사 | `dinner_status` | 매일 | 식사 규칙성 트렌드 |
| 채소 섭취 | `vegetable_intake_level` | 매일 | FINDRISC #5 연동 |
| 식사 구성 | `meal_balance_level` | 매일 | 식단 품질 점수 트렌드 |
| 당 음료/간식 | `sweetdrink_level` | 매일 | 당 섭취 추이 |
| 운동 여부·종류·시간 | `exercise_done`, `exercise_type`, `exercise_minutes` | 매일 | 주간 합산 → 150분 목표 달성률 |
| 산책 | `walk_done` | 매일 | 일별 산책 실천률 |
| 수분 섭취 | `water_cups` | 매일 | 일별 수분 섭취 기록 |
| 야식 | `nightsnack_level` | 매일 | 야식 빈도 추이 |
| 기분/스트레스 | `mood_level` | 2-3일 주기 | 주간 평균 스트레스 추이 |
| 음주 | `alcohol_today`, `alcohol_amount_level` | 2-3일 주기 | 음주 빈도/량 추이 |
| 허리둘레 | (periodic_measurements) | 월 1회 | 월별 변화 추이 |
| 혈압 | (periodic_measurements) | 주 1-2회 | 혈압 변화 그래프 |
| 공복혈당 / HbA1c / 콜레스테롤 | (periodic_measurements) | 3-12개월마다 | 분기별 검사 결과 이력 |

**출처 추적**: 각 필드마다 `_source` 필드가 있어 AI 채팅(`chat`)/직접입력(`direct`)/소급입력(`backfill`)을 구분

## 3. 생활습관 챌린지

챌린지는 "오늘 목표 달성했나요?"를 체크하는 기능. 매일 수집 데이터를 그대로 재활용.

| 챌린지 항목 | 입력 질문 | 목표 기준 |
|------------|---------|---------|
| 운동 챌린지 | 오늘 운동 시간/종류 | 주 150분 이상 |
| 채소 챌린지 | 오늘 채소 먹었나요? | 매일 섭취 |
| 수면 챌린지 | 오늘 수면 시간·질 | 7-8시간/밤 |
| 음수 챌린지 | 오늘 물 몇 컵? | 하루 8컵 (1.6-2L) |
| 복약 챌린지 | 오늘 약 드셨나요? | 연속 복약 유지 |
| 야식 자제 챌린지 | 야식 드셨나요? | 야식 없음 |
| 스트레스 관리 챌린지 | 오늘 기분 어떠셨나요? | 명상/이완 활동 실천 |
| 걸음 수 챌린지 | 걸음 수 (자동 수집) | 하루 7,000보 이상 |

> **핵심**: 3개 기능에서 수집하는 질문들이 대부분 겹친다.
> "수면 시간" 하나를 입력하면 예측 모델 + 대시보드 + 챌린지 3곳에 동시에 활용된다.

## 4. DB 모델이 바뀌는 경우

DB 모델 변경 = **엑셀 표의 열(column)을 추가/삭제/수정하는 것**

### 케이스 1: 수집 항목이 늘거나 줄 때

가장 흔한 경우. 새 질문 = 새 열(column) 추가

```
예시: "야식 시간도 기록하자"
→ daily_logs 테이블에 night_snack_time 열 추가
```

---

### 케이스 2: 예측 모델이 Phase 업그레이드될 때

| Phase | 새로 필요한 데이터 | DB 변화 |
|-------|----------------|--------|
| Phase 1 (MVP) | FINDRISC 8개 항목 | 기본 건강기록 테이블 |
| Phase 2 | 혈압·혈당 추가 | 검사결과 테이블 추가 |
| Phase 3 | 혈액검사 전체 (콜레스테롤, GGT 등) | 검사항목 테이블 확장 |

---

### 케이스 3: 챌린지 설계가 바뀔 때

```
예시: 단순 체크 → streak(연속 달성) 카운트 방식으로 변경
→ challenge_logs 테이블에 streak_count, last_completed_at 열 추가
```

---

### 케이스 4: 사용자 상태 관리 기능 추가 시

5단계 참여 상태(ACTIVE → MODERATE → LOW → DORMANT → HIBERNATING)를 DB에 저장하려면

```
→ users 테이블에 engagement_status 열 추가
```

---

### 요약

```
DB 모델이 바뀌는 경우
├── 1. 새 질문 추가/삭제       → 열(column) 추가/삭제
├── 2. 예측 모델 Phase 업그레이드 → 새 테이블 또는 열 추가
├── 3. 챌린지 방식 변경        → 챌린지 관련 테이블 수정
└── 4. 사용자 상태 관리 추가   → users 테이블 확장
```
