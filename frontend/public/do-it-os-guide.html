머릿속이 복잡한가요?
Do it OS에 내려놓으세요.

Do it OS는 "머릿속에서 뱅뱅 도는 생각"을 밖으로 꺼내서 제자리로 보내주는 도구예요. 이 가이드는 처음 사용자도 순서대로 따라가면 바로 쓸 수 있게 준비됐습니다. 심리학·인지과학 근거부터 실제 화면 와이어프레임까지 한 번에 담았어요.

① 생각 쏟기 → ② 명료화 → ③ 카테고리 관리 → ④ 자기 전 정리 → ⑤ 다음 날 순환
🧠

먼저 "왜 쓰는지" 심층 — 5가지 과학적 근거

도구가 아니라 사고 설계. 이 5개 연구가 Do it OS 전체 흐름을 만들어요.
🔬 "왜 그렇게 하는가?"를 알면 "그렇게 안 하면 왜 힘든가?"가 저절로 보여요.
근거 1

자이가르닉 효과 (Zeigarnik Effect)

1927 · Bluma Zeigarnik (리투아니아 심리학자)

"완결되지 않은 과제가 완결된 과제보다 2배 더 기억에 남는다"

빈 카페에서 웨이터 관찰 연구에서 출발. 아직 내지 않은 주문은 또렷하게 기억하지만 계산이 끝난 주문은 거의 잊어버림. 현대 인지과학에서는 미완결 "열린 루프(open loop)"가 작업 기억을 지속 점유하면서 인지 부하를 누적시켜 수면 방해와 주의력 저하를 일으킨다고 해석해요.

Do it OS 처방

자기 전 정리 리츄얼에서 남은 생각을 모두 버리기 / 내일 / 대기 중 하나로 결정 → 열린 루프 0으로 수렴 → 뇌가 쉬게 함.

근거 2

GTD: Getting Things Done

2001 · David Allen (생산성 컨설턴트)

"The mind is for having ideas, not holding them.
(머리는 아이디어가 떠오르는 곳이지, 저장하는 곳이 아니다)"

Allen의 원본 5단계는 Capture → Clarify → Organize → Reflect → Engage. 30년간 실무 컨설팅에서 검증된 모델이지만, 단계가 많고 용어가 낯설어 처음 사용자에게 학습 장벽이 있어요.

Do it OS 처방

5단계를 3단계(쏟기 → 명료화 → 관리)로 단순화. Reflect·Engage는 자기 전 정리 리츄얼과 다음 날 순환 구조로 자연스럽게 흡수 → 초보자 학습 부담 낮춤.

근거 3

작업 기억 한계 (Working Memory Capacity)

1956 Miller → 2001 Cowan (심리학자)

"Magical Number Seven, Plus or Minus Two → 실제로는 4±1개"

Miller의 유명한 7±2 규칙을 Cowan이 2001년 재검증한 결과, 청킹·시연·장기기억 보조 없이 순수 작업기억 용량은 4±1에 불과했어요. 열린 루프가 7개를 넘으면 전두엽 과부하로 "결정 피로(decision fatigue)"가 생겨 의사결정 품질이 급격히 떨어져요.

Do it OS 처방

머릿속 대신 외부 시스템에 위임 → 작업 기억 용량 해방. "다 적었으니 잊어도 되는" 상태 확보가 인지 자원 해방의 핵심.

근거 4

외부화 효과 (Externalization)

2010 · David Kirsh (UC San Diego 인지과학)

"적는 행위 자체가 사고를 재구성한다 — 단순 저장이 아니다"

Kirsh의 연구는 "외부 표상"이 단순히 메모리를 대체하는 것이 아니라, 사고 과정을 직접 재구성한다는 것을 보여줘요. "분류"는 인지적 처리 — 카테고리 선택 자체가 생각을 명확하게 만들고, 손가락으로 칩을 누르는 물리적 동작은 결정의 물리적 각인이 됩니다.

Do it OS 처방

인라인 명료화 패널로 가장 짧은 동선에서 외부화 완성. 카드 내부에서 칩 → 게이트 → 저장까지 한 화면으로 → 마찰 최소 + 외부화 품질 최대.

근거 5

2분 규칙 + 결정 피로

2001 Allen · 2007 Baumeister (의지력 연구)

"2분 안에 끝날 일은 지금 바로" · "결정할수록 전두엽이 지친다"

GTD의 2분 규칙은 "저장 오버헤드가 실행 시간보다 크면 바로 해라"는 원칙. Baumeister의 2007년 의지력 연구는 하루 동안 내리는 결정이 쌓일수록 저녁에는 의지력이 고갈되어 판단이 흐려진다는 것을 증명했어요.

Do it OS 처방

"실행 가능한가?" 단일 게이트로 결정 압력 최소화. 7개 카테고리 중 고르기 전에 단 하나의 질문만 통과 → 저녁까지 의지력 보존.

1

대시보드에 들어가기

어디로 들어가고, 뭐가 보이나요?
🗺 "Do it OS는 왼쪽 사이드바에 있어요."
사이드바 구조 · Do it OS는 7개 하위 메뉴를 가진 상위 메뉴

대시보드에 뭐가 보여요?

① 미분류·대기중·언젠가 통계 한 줄 / ② 생각 Inbox (아직 분류 안 된 메모 미리보기) / ③ 오늘 일정 (오늘 날짜로 지정된 일정 상위 3개) / ④ 정리 결과 보드 (5개 주요 카테고리별 저장된 항목 프리뷰) / ⑤ 액션 스트립 5장 (프로젝트·노트·건강 단서·저장 원칙·자기 전 정리 바로가기) / ⑥ 🌙 "내일 하기로" 배지 섹션 (어제 골라둔 항목들) / ⑦ 저녁 CTA (21시 이후 조건부) / ⑧ 최근 사용 흐름 힌트

어디서부터 시작해야 해요?

지금이 낮이면 → Step 2 "생각 쏟기"부터. 지금이 밤(21시 이후)이면 → Step 5 "자기 전 정리"로 바로 가도 돼요. 대시보드 자체는 "현재 상태 확인" 용도예요.

첫 사용이면 뭐부터 해볼까요?

3분 투어 추천: ① 대시보드에서 현재 상태 훑어보기 → ② "생각 쏟기"에 머릿속 3~5개 던지기 → ③ "정리 명료화"에서 첫 카드 하나만 분류해보기. 이 세 가지만 해도 전체 흐름이 감 잡혀요.

2

생각 쏟기 — 판단 없이 마구 꺼내기

모든 흐름의 출발점. "나중에 어디 갈지"는 전혀 신경 쓰지 마세요.
🧠 "머릿속에 맴도는 것"을 모두 종이 조각에 던진다고 상상하세요.
  1. 사이드바에서 🧠 생각 쏟기 클릭 → /app/do-it-os/thinking 화면으로 이동
  2. 화면 하단의 입력창에 생각을 한 문장씩 자유롭게 적기 (판단·분류 금지)
  3. Enter 또는 "쏟아내기" 버튼으로 저장 → 화면 중앙 캔버스에 파스텔 메모 카드로 올라옴
  4. 원하는 만큼 반복. 이 단계에서는 "많이 꺼낼수록 좋다"는 게 유일한 규칙
캔버스에 파스텔 메모 카드 5색(크림·스톤·민트·라벤더·블러시)이 랜덤 위치·회전으로 배치됨

"이거 할 일인지 노트인지 모르겠어요"

정답입니다. 이 단계에서는 판단을 안 하는 게 핵심이에요. 판단은 Step 3 (정리 명료화)에서 한꺼번에 해요. 이 단계에서 고민하면 생각이 더 안 나와요.

얼마나 많이 쏟아야 돼요?

보통 한 번에 5~15개. 머릿속에서 뱅뱅 돌던 게 다 나왔다 싶을 때까지. "더 없나?" 자문해보고 그래도 안 나오면 충분한 거예요.

쏟은 건 어디에 저장돼요?

브라우저의 localStorage에 저장돼요 (서버에 안 감). 키 이름은 danaa_doit_thoughts_v1. 같은 브라우저에서만 보여요.

⏩ 쏟기 다음은?
이제 꺼낸 생각들을 제자리로 보내주는 정리 명료화 단계입니다. 다음 Step 3으로 이동하세요.
3

정리 명료화 — 카드 안에서 바로 분류

Do it OS 핵심 기능. 각 생각을 7개 카테고리 중 하나로 보내요.
✅ "생각 쏟기에서 나온 메모를 하나씩 제자리로 보내는 단계."
  1. 사이드바 ✅ 정리 명료화 클릭 → /app/do-it-os/classify
  2. 미분류 메모가 한 장씩 카드로 나열됨. 카드에 7개 칩이 달려있어요
  3. 카드의 분류 칩을 클릭하면 → 같은 카드 안에서 인라인 패널이 펼쳐짐 ⭐ (오른쪽 드로어 아님!)
  4. 패널 안에서 "실행 가능한가요?" 게이트가 먼저 뜸 → 예/아니오 선택
  5. 카테고리별로 추가 입력 (날짜·시간·맡긴 사람 등) 후 저장

📌 7개 분류 카테고리

할 일 일정 프로젝트 노트 건강 단서 대기 중 언젠가

앞 5개(할 일·일정·프로젝트·노트·건강 단서)는 주요 카테고리, 뒤 2개(대기 중·언젠가)는 보조 분류(사이드바 메뉴 없음, 명료화 페이지 하단 섹션에서 관리)

칩 클릭 → 같은 카드 내부에서 게이트 + 입력 + 저장이 다 이뤄짐 (오른쪽 드로어 없음!)

🚪 게이트 분기: "실행 가능한가요?"

예 (실행 가능)

→ 해당 카테고리로 저장

todo: "2분 안에 끝날 일이면 지금 바로 해보세요" 힌트 + 저장
schedule: 날짜 + 시간 입력
project: 즉시 저장 (토스트에 "상세에서 이어쓰기" 링크)
waiting: "누구에게 맡겼나요?" 입력

아니오 (참고만)

→ 3가지 선택지

노트로 보관: 참고 자료로 저장 (나중에 볼 수 있음)
언젠가: 지금은 안 함, 나중에 다시 볼지 결정
버리기: 토스트에 "되돌리기" 5초 표시 (복구 가능)

⏭ 게이트를 건너뛰는 카테고리
note · health · someday는 이미 "참고 정보"에 해당해서 게이트 없이 바로 저장돼요. 칩 하나 → 저장 버튼 → 끝.
4

카테고리별 관리 — 상세 페이지에서 이어 쓰기

분류가 끝나면 각 카테고리 페이지에서 이어서 편집 · 완료 체크 · 분류 해제까지.
📂 Phase 7.1~7.2 업데이트: 체크박스 · 완료 폴더블 · 분류 해제 · 되돌리기 토스트 도입
📅 일정Phase 7.1

좌측 체크박스 + 완료 폴더블

각 일정 카드 좌측에 체크박스. 체크 시 본문에 strike 처리 + "완료된 일정 N개 ▾" foldable 섹션으로 자동 이동. 폴더블을 펼치면 과거 완료 기록이 히스토리로 남아요. 체크 해제하면 활성 목록으로 복귀.

✅ 할 일Phase 7.1

체크박스 + 분류 해제 버튼

일정과 동일한 체크박스 동작. 카드 우측에 "분류 해제" 버튼 (이전 이름: "Inbox로"). 호버 시 "미분류로 돌려보내기" 툴팁. 클릭하면 5초 되돌리기 토스트.Phase 7.2

📂 프로젝트

연결된 다음 행동 + 히스토리

프로젝트 카드 클릭 → 상세. 설명·상태(진행중/잠시중단/완료) 편집, 10가지 다음 행동 프리셋 카드로 하위 할 일 추가. ⭐ "연결된 다음 행동" 섹션에 완료 항목도 history view로 남음.

📝 노트

제목 + 본문 + 자동 저장

노트 카드 클릭 → 상세. 제목·본문 textarea. 0.7초 입력 멈춤 감지 → 자동 저장. "저장됨" 표시. 상세 페이지에도 "분류 해제" 버튼 있음.

⏳ 대기 중

명료화 페이지 하단 보조 섹션

사이드바에 별도 메뉴 없음. 정리 명료화 페이지 하단의 "보조 분류" 섹션에서 관리. "누구에게 맡겼는지" 메모와 함께 표시. "분류 해제" 버튼도 동일하게 제공.

☁️ 언젠가

"보류함"

지금은 안 하지만 완전히 버리긴 애매한 것. 나중에 다시 생각날 때 복구 가능. 대시보드 통계 1줄에 수량 표시. 마음이 바뀌면 "분류 해제"로 미분류로 되돌리기 가능.

🩺 건강 단서

몸·마음 관련 메모 (DB 격리)

"무릎이 뻐근함", "수면이 얕았음" 같은 자기 관찰 기록. 다나아 건강 DB와 자동 연결되지 않음 (개인 메모로만 격리됨). 챌린지·리포트에도 전달되지 않아요.

모든 카테고리 카드·상세 페이지에 "분류 해제" 버튼이 있어요. 클릭하면:
  • category를 null로 초기화 (미분류 상태)
  • scheduledDate, scheduledTime, completedAt 필드도 초기화
  • 화면 하단에 5초 되돌리기 토스트 표시 → 클릭하면 원래 카테고리로 복구
  • 정리 명료화 페이지의 미분류 카드로 복귀
✨ Tip: 프로젝트 ↔ 다음 행동 연결
프로젝트 상세에서 "다음 행동 카드 그리드"에서 새 할 일을 만들면 자동으로 projectLinkId가 저장돼요. 그 할 일은 분류 후 프로젝트 상세의 "연결된 다음 행동" 섹션에 자동 표시. 완료 체크 후에도 히스토리로 남아 "무엇을 해왔는지" 한눈에 보여요.

🎨 Phase 7.3 · Step 3 호버 색상 구분 Phase 7.3
Step 3에서 각 메모 옆 3개 버튼이 호버 시 명확한 의미 전달 색으로 바뀝니다:
  • 🗑 버리기: 빨강 호버 (결정의 무게)
  • ⏸ 내일도 고민: 노랑 호버 (보류 · 다음 날 미분류로 복귀)
  • 👥 대기중: 보라 호버 (맡김 · 다른 사람 기다리기)
🔧 Phase 7.3.1 버그 수정 Phase 7.3.1
Step 1에서 쏟은 메모가 Step 2에 즉시 반영됩니다. 이전엔 Step 1에서 추가 생각을 적으면 Step 2로 넘어갔을 때 미분류 섹션에 보이지 않았는데, 지금은 상태가 공유되어 바로 나타나요. 생각 → 분류 흐름이 끊기지 않습니다.

"버리기"는 진짜 삭제되나요?

아니요, 복구 가능해요. 버리기 클릭 → 토스트에 "버렸어요 · [되돌리기]" 5초 표시. 클릭하면 복구. 토스트가 사라진 후에도 내부적으로는 discardedAt 타임스탬프만 기록되고, 어느 카테고리 목록에도 안 보이도록 숨겨집니다 (물리적 삭제 아님).

꼭 21시 이후에만 해야 되나요?

아니요. 아무 때나 사이드바 → 자기 전 정리 클릭하면 돼요. "21시 이후 CTA"는 대시보드에 은근한 알림으로 나오는 것이고, 리츄얼 페이지 자체는 언제나 접근 가능.

"내일도 고민"을 고르면 어떻게 돼요?

endOfDay.action='keep'으로 기록만 남고, 카테고리는 유지됩니다. 다음 날 명료화 페이지 미분류 섹션에서 다시 볼 수 있어요. 오늘 결정 안 해도 괜찮다는 느슨한 처리.

💡 하루를 잘 마무리하는 비결
리츄얼의 진짜 목적은 "일을 다 끝내는 것"이 아니에요. "내일 뭐부터 할지 결정해두는 것"이에요. 뇌는 "결정이 끝난 것"에는 매달리지 않아요. 1~3개만 골라놓으면 그게 핵심.

자주 묻는 질문

실수했을 때·잘 모를 때 참고하세요
"분류 해제"가 뭔가요? 어디로 가나요?
"분류 해제"는 카드를 미분류 상태로 되돌리는 기능이에요. 이전엔 "Inbox로"라는 이름이었는데 헷갈려서 Phase 7.1부터 "분류 해제"로 바뀌었어요.

동작:
  • category = null로 초기화
  • scheduledDate, scheduledTime, completedAt 함께 초기화
  • → 정리 명료화 페이지의 미분류 카드로 복귀
  • 화면 하단에 5초 되돌리기 토스트 표시 (클릭하면 원래 카테고리로 복구)
모든 카테고리 카드·상세 페이지 우측에 버튼이 있어요. 마음이 바뀌면 언제든 쓰세요.
"내일도 고민"을 누르면 어떻게 돼요?
endOfDay.action='keep'으로 기록만 남고, 항목의 카테고리는 그대로 유지됩니다. 실제 효과:
  • 자기 전 정리 Step 3에서는 해당 항목이 사라져요
  • 다음 날 정리 명료화 페이지 미분류 섹션에서 다시 볼 수 있어요
  • 결정을 미뤄도 데이터가 손실되지 않아요
"버리기"랑 다르게 결정 연기일 뿐, 삭제가 아니에요.
완료 체크가 영구 삭제인가요?
아니요, 데이터는 그대로 보존돼요. 할 일·일정 카드 좌측 체크박스를 누르면:
  • completedAt에 타임스탬프 기록
  • 본문 strike 처리
  • 목록에서 "완료된 일 N개 ▾" foldable 섹션으로 이동 (숨김 아님)
  • 폴더블을 펼치면 과거 완료 항목 모두 확인 가능
  • 다시 체크 해제하면 활성 목록으로 복귀
프로젝트 상세의 "연결된 다음 행동"에서도 완료 항목이 history view로 남아 있어요.
"대기 중"과 "언젠가" 차이가 뭐예요?
대기 중 (waiting): 다른 사람에게 맡긴 상태. 내가 안 해도 되지만 결과를 기다려야 함. "누구에게 맡겼나요?" 입력 필드가 있음. 예: "개발팀 검토 대기", "아버지 답장 대기".

언젠가 (someday): 지금은 안 함, 나중에 다시 볼지 결정. 기한·담당자 없음. 예: "언젠가 읽을 책", "나중에 해볼 운동".
데이터는 어디에 저장되나요? 서버에 가나요?
브라우저 로컬스토리지에만 저장돼요. 서버 호출 0. 키 이름은 danaa_doit_thoughts_v1. 다른 기기·다른 브라우저에서는 동기화되지 않아요. 개인 메모용.

⚠️ 건강 단서 카테고리도 다나아 건강 기록 DB와 자동 연결되지 않음. 개인 관찰 메모로만 격리 보관. 챌린지·리포트·AI 채팅에 자동으로 전달되지 않아요. 원한다면 직접 복사해서 건강 기록에 입력해야 해요 (프라이빗 디폴트).
대기 중·언젠가 항목은 어디서 봐요?
사이드바에 별도 메뉴가 없어요 (과밀 방지). 정리 명료화 페이지의 하단 "보조 분류" 섹션에서 2컬럼으로 볼 수 있어요. 그리고 대시보드 통계 1줄에 수량이 표시돼요.
ClassifyInlinePanel이 이전 Phase 6와 뭐가 달라요?
Phase 6: 칩 클릭 → 화면 오른쪽 끝에 슬라이드 드로어 → 저장 버튼도 드로어 안 → 마우스 이동 大.

Phase 7 (현재): 칩 클릭 → 같은 카드 내부에서 인라인 패널이 max-height 애니메이션으로 확장 → 저장 버튼도 카드 안 → 마우스 이동 최소. 여러 카드를 한 화면에서 빠르게 처리하는 흐름에 맞춰 개선됨. 근거: Kirsh 2010 외부화 효과 (짧은 동선일수록 외부화 품질 상승).
2분 규칙이 뭐예요?
GTD(Getting Things Done) 기법의 핵심 규칙. "2분 안에 끝낼 수 있는 일이면 저장하지 말고 지금 바로 해라"라는 원칙이에요. 저장했다가 나중에 처리하면 그 자체가 오버헤드. Do it OS는 "할 일" 분류 시 이 힌트를 1줄로 표시해서 바로 하도록 유도해요.
프로젝트에 "연결된 다음 행동"은 어떻게 추가해요?
프로젝트 상세 페이지 → "다음 행동 카드 그리드" 섹션에서 10개 프리셋(할 일·일정·노트·회의·연락·조사·결정·피드백·리뷰·마감) 중 하나 클릭 → 자동으로 새 할 일 생성되고 projectLinkId가 이 프로젝트로 설정됨. 그 후 해당 할 일은 정리 명료화에서 분류하면 프로젝트 상세에 자동 표시. 완료 체크 후에도 history view로 남아요.
자주 쓰는 단축키는?
  • Enter: 생각 쏟기에서 메모 저장
  • ESC: 인라인 명료화 패널 닫기 / 리츄얼 중단 (데이터 변경 0)
  • Tab: 폼 내부 이동
🌙 "내일 하기로" 배지는 언제 사라져요?
자기 전 정리 Step 2에서 체크한 항목은 다음 날 하루 동안 대시보드·명료화·카테고리·정리 결과 보드 4곳에 🌙 배지로 표시돼요. 사라지는 조건:
  • 완료 체크를 하면 배지 자동 제거
  • 다음 자기 전 정리 리츄얼을 시작하면 지난 배지는 자동 초기화
  • 카드를 "분류 해제"하면 배지도 함께 사라짐
