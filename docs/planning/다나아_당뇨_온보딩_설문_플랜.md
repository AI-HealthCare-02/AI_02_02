# 다나아(DA-NA-A) 당뇨 카테고리 온보딩 설문 플랜

---

## 설계 근거

본 플랜은 검증된 3가지 당뇨 위험도 평가 도구를 기반으로 설계했다.

| 도구 | 핵심 항목 | 특징 |
|------|-----------|------|
| FINDRISC (핀란드) | 나이, BMI, 허리둘레, 혈압약, 고혈당 이력, 운동, 식습관 (8문항) | 글로벌 표준, 혈액검사 불필요 |
| ADA Risk Test (미국) | 나이, 성별, 임신성당뇨, 가족력, 고혈압, 운동, BMI (7문항) | 60초 완료 목표 |
| Korean Diabetes Risk Score | 나이, 가족력, 고혈압, 허리둘레, 흡연, 음주 (6문항) | 한국인 체형 기준 반영 |

**핵심 발견:** 나이 + BMI/허리둘레 + 가족력 + 운동량, 이 4가지만으로 예측력의 약 80%를 확보할 수 있다.

**이탈률 데이터:**
- 1~3문항: 완료율 80%+
- 4~8문항: 완료율 65%
- 9~14문항: 완료율 56%
- 15문항 이상: 완료율 42% (문항 1개 추가당 5~10% 하락)

→ **목표: 핵심 7스크린 + 조건부 2~3스크린 = 체감 10스크린 이내, 2분 이내 완료**

---

## 사용자 분류 체계 (콜드스타트 3분기)

당뇨 카테고리를 선택한 사용자는 첫 질문의 답변으로 3개 그룹으로 나뉜다.

```
[당뇨 선택]
    │
    ├─ A. 관리형: "당뇨 진단을 받았어요"
    │     → 치료 정보 + 수치 입력 화면 추가
    │
    ├─ B. 주의형: "전당뇨(경계)라고 들었어요"
    │     → 수치 입력 간소화, 생활습관 중심
    │
    └─ C. 예방형: "가족력 있음 / 궁금해서 / 건강관리 목적"
          → 위험인자 중심, 치료 질문 스킵
```

이 분기에 따라 뒤의 설문이 달라지므로, 사용자는 자기에게 불필요한 질문을 만나지 않는다.

## 전체 설문 플로우 (스크린별 상세)

### ━━━ Phase 1: 누구인지 파악 (3스크린, 필수) ━━━

#### Screen 1 — 당뇨와의 관계
> 화면 상단 텍스트: "당뇨와 어떤 관계인가요?"

| 선택지 (단일선택, 카드형 버튼) |  분기 |
|---|---|
| 당뇨 진단을 받았어요 | → **A. 관리형** |
| 전당뇨(경계)라고 들었어요 | → **B. 주의형** |
| 가족 중에 당뇨가 있어요 | → **C. 예방형** |
| 그냥 궁금해서요 | → **C. 예방형** |
| 건강관리 차원에서 | → **C. 예방형** |

> UX 팁: 각 카드에 한 줄 보조 설명 추가 (예: "의사에게 당뇨병 진단을 받은 적이 있다면")

---

#### Screen 2 — 기본 프로필
> 화면 상단 텍스트: "기본 정보를 알려주세요"

**2-1. 성별** (단일선택, 가로 배치 2버튼)
- 남성
- 여성

**2-2. 연령대** (단일선택, 가로 그리드)
- 20대 | 30대 | 40대 | 50대 | 60대 이상

> 설계 이유: 한국 당뇨 위험도 점수에서 나이(45세 이상 = 3점)가 최대 배점. 정확한 생년월일 대신 연령대로 부담을 줄임.

---

#### Screen 3 — 체형 정보
> 화면 상단 텍스트: "체형을 알려주세요"

**입력 방식: 슬라이더 2개** (텍스트 입력 없이 드래그만)
- 키: 140cm ──●── 200cm (5cm 단위 스냅)
- 몸무게: 40kg ──●── 150kg (5kg 단위 스냅)

> 시스템이 BMI를 자동 계산하여 화면 하단에 실시간 표시
> 예: "BMI 24.2 — 정상 범위"

**대안 (슬라이더 부담 시):** 체형 일러스트 5단계 선택
- 마른 편 | 보통 | 약간 과체중 | 과체중 | 비만

> 설계 이유: BMI는 FINDRISC, ADA, Korean Score 모두에서 핵심 변수. 슬라이더는 타이핑 없이 정보 수집 가능.

---

### ━━━ Phase 2: 핵심 위험인자 (2~4스크린, 조건부) ━━━

#### Screen 4 — 가족력
> 화면 상단 텍스트: "가까운 가족 중 당뇨가 있나요?"

| 선택지 (단일선택) |
|---|
| 부모님 중 있어요 |
| 형제·자매 중 있어요 |
| 부모 + 형제 모두 있어요 |
| 없어요 |
| 잘 모르겠어요 |

> 설계 이유: 가족력은 ADA, Korean Score 모두 1점 배점. "잘 모르겠어요" 옵션으로 이탈 방지.

---

#### Screen 5 — 동반 상태 확인
> 화면 상단 텍스트: "해당하는 것을 모두 골라주세요"

| 선택지 (복수선택, 토글 칩) |
|---|
| 고혈압 (또는 혈압약 복용 중) |
| 고지혈증 |
| 고혈당 판정 받은 적 있음 |
| 임신성 당뇨 경험 (여성만 표시) |
| 해당 없음 |

> 설계 이유: FINDRISC에서 "고혈당 이력" 단독 5점(최대 배점), "혈압약 복용" 2점. 복수선택으로 한 화면에 처리.

---

#### Screen 6 — [A. 관리형만] 현재 치료 상황
> 화면 상단 텍스트: "현재 어떤 관리를 하고 계세요?"

| 선택지 (복수선택) |
|---|
| 식이요법 / 생활습관 관리만 |
| 경구 혈당강하제 복용 |
| 인슐린 주사 |
| 기타 약물 |
| 특별히 관리하지 않고 있어요 |

> B, C 그룹은 이 화면을 건너뜀

---

#### Screen 7 — [A, B만] 최근 수치 (선택적)
> 화면 상단 텍스트: "최근 검사 수치를 알고 있다면 알려주세요"
> 보조 텍스트: "몰라도 괜찮아요! 나중에 입력할 수 있어요"

**7-1. 당화혈색소 (HbA1c)** — 범위 선택 버튼
- 5.7% 미만 (정상)
- 5.7~6.4% (전당뇨)
- 6.5~7.0%
- 7.0~8.0%
- 8.0% 이상
- **모르겠어요** ← 기본 하이라이트

**7-2. 공복혈당** — 범위 선택 버튼
- 100 미만 (정상)
- 100~125 (경계)
- 126 이상 (당뇨)
- **모르겠어요** ← 기본 하이라이트

> C 그룹은 이 화면을 건너뜀
> "모르겠어요"를 기본 선택으로 두어 부담을 최소화

---

### ━━━ Phase 3: 생활습관 기본 (3스크린, 필수) ━━━

#### Screen 8 — 운동 빈도
> 화면 상단 텍스트: "일주일에 운동을 얼마나 하시나요?"
> 보조 텍스트: "걷기, 계단 오르기 등 가벼운 활동도 포함"

| 선택지 (단일선택, 아이콘 카드) |
|---|
| 거의 안 해요 |
| 주 1~2회 |
| 주 3~4회 |
| 주 5회 이상 |

> 설계 이유: FINDRISC "주 4시간 이상 운동 여부" 2점, ADA "주 대부분 30분 운동" 1점. 두 기준 모두 커버.

---

#### Screen 9 — 식습관 핵심
> 화면 상단 텍스트: "식습관을 골라주세요"

| 선택지 (복수선택, 토글 칩) |
|---|
| 밥·빵·면 위주로 먹어요 (탄수화물 과다) |
| 단 음료를 자주 마셔요 |
| 야식을 자주 먹어요 |
| 채소·과일을 매일 먹어요 |
| 식사가 불규칙해요 |
| 특별히 해당 없음 |

> 설계 이유: FINDRISC "매일 채소/과일 섭취 여부" 1점. 당뇨 관리에서 탄수화물·당류 섭취 패턴이 핵심.

---

#### Screen 10 — 수면 + 음주·흡연
> 화면 상단 텍스트: "생활 패턴을 알려주세요"

**10-1. 평균 수면시간** (단일선택, 가로)
- 5시간 미만 | 5~6시간 | 7~8시간 | 9시간 이상

**10-2. 음주** (단일선택)
- 안 마셔요 | 가끔 (월 1~3회) | 자주 (주 1회+) | 매일

**10-3. 흡연** (단일선택)
- 안 피워요 | 과거 흡연 (현재 금연) | 현재 흡연 중

> 설계 이유: Korean Score에서 흡연 1점, 음주 최대 3점. 수면은 인슐린 저항성과 밀접.
> 3가지를 한 화면에 묶되, 각 항목을 세로로 분리 배치하여 시각적 과부하 방지.

---

### ━━━ Phase 4: 개인화 설정 (1스크린, 필수) ━━━

#### Screen 11 — 목표 설정
> 화면 상단 텍스트: "다나아에서 가장 하고 싶은 것은?"

| 선택지 (복수선택, 아이콘 카드) |

| 혈당 / 건강수치 추적하기 |

| 운동 습관 만들기 |
| 체중 관리하기 |
| 전체 다 관심 있어요 |

## 이탈 방지 UX 전략 7가지

### 1. 프로그레스 바 + 남은 시간 표시
- 화면 상단에 얇은 바: ████████░░░░ 7/10
- 첫 화면에 "약 2분이면 끝나요" 표시

### 2. 매 화면 "왜 물어보는지" 한 줄 설명
- Screen 3 (체형): "체질량지수(BMI)는 당뇨 위험 예측의 핵심 지표예요"
- Screen 4 (가족력): "부모의 당뇨 이력이 있으면 위험도가 2~3배 높아져요"

### 3. "모르겠어요 / 건너뛰기" 항상 제공
- 모든 선택형 질문에 "모르겠어요" 옵션 포함
- 선택적 입력(수치 등)은 "나중에 입력할게요" 버튼
- 어떤 질문도 막히는 느낌을 주지 않음

### 4. 중간 보상 메시지 (Phase 전환 시)
- Phase 1 → 2 전환: "기본 프로필 완성! 이제 절반 왔어요 💪"
- Phase 2 → 3 전환: "거의 다 됐어요! 마지막 생활습관만 남았어요"

### 5. 한 화면 = 한 질문 원칙 (예외: Screen 10만 3개 묶음)
- 인지 부하를 최소화
- Screen 10은 모두 "선택만 하면 되는" 간단한 항목이라 묶어도 부담 적음

### 6. 클릭 전용 UI — 텍스트 입력 0
- 모든 답변: 버튼 탭 / 슬라이더 드래그 / 토글 칩
- 키보드가 올라오는 순간이 없음
- 슬라이더에는 양쪽 끝 라벨 + 현재 값 실시간 표시

### 7. 이탈 시 저장 + 복귀 유도
- 중간에 나가도 답변한 데이터는 자동 저장
- 다시 접속 시: "이전에 하시던 설문을 이어서 할까요?" 팝업
- 완료하지 않아도 수집된 데이터만으로 기본 대시보드 제공

## 위험도 점수 산출 (내부 로직, 사용자에게 비노출)

온보딩 데이터로 **한국형 + FINDRISC 하이브리드 점수**를 산출한다.

| 항목 | 점수 범위 | 근거 |
|------|:---------:|------|
| 연령 (45세+ = 고위험) | 0~4 | FINDRISC |
| BMI (25이상 주의, 30이상 위험) | 0~3 | FINDRISC + ADA |
| 가족력 | 0~1 | ADA + Korean |
| 고혈압 / 혈압약 | 0~2 | FINDRISC |
| 고혈당 이력 | 0~5 | FINDRISC (최대 배점) |
| 운동 부족 | 0~2 | FINDRISC |
| 식습관 (채소/과일 미섭취) | 0~1 | FINDRISC |
| 흡연 | 0~1 | Korean Score |
| 음주 (과다) | 0~3 | Korean Score |
| **합계** | **0~22** | |

위험 등급:
- 0~7: 낮음 (초록) → "좋은 습관을 유지하고 있어요"
- 8~14: 보통 (노랑) → "몇 가지 생활습관 개선이 도움될 수 있어요"
- 15~18: 높음 (주황) → "정기 검진과 생활습관 관리가 필요해요"
