# 다나아(DA-NA-A) 프로젝트 브리핑

> 이 문서는 VS Code CLI에서 개발을 시작할 때, AI 또는 개발자가 프로젝트의 전체 맥락을 빠르게 파악할 수 있도록 정리한 것입니다.

---

## 1. 서비스 정의

### 한 줄 정의
사용자가 이미 자주 쓰는 AI 채팅 환경 안에서, 원래 답변 흐름을 끊지 않고 짧은 건강 질문을 자연스럽게 덧붙여 생활습관 데이터를 쌓고, 이를 바탕으로 만성질환 위험 예측·추적 대시보드·행동 유도형 챌린지를 제공하는 웹 서비스.

### 팀 공통 정의문
- 대상: 당뇨·고혈압 등 만성질환 위험이 있거나, 아직 위험을 충분히 인지하지 못한 성인 사용자
- 핵심 아이디어: ChatGPT, Claude, Gemini 같은 AI 채팅을 평소처럼 이용하는 동안, 답변 하단에 건강 질문을 짧고 자연스럽게 추가하여, 환경을 바꾸지 않고도 건강정보를 축적
- 축적된 정보의 활용: 만성질환 예측 모델, 추적 대시보드, 생활습관 개선 챌린지
- 현재 초점: **당뇨**를 중심으로 먼저 구현

### 핵심 문제 정의
기존 건강 앱은 사용자가 일부러 앱을 켜고 기록해야 해서 금방 이탈. 다나아는 건강기록의 진입장벽과 지속 사용의 어려움을 줄이는 데 초점.

## 2. 현재 구현 범위 (이번 개발)

### 만들어야 할 것
1. **카테고리 선택 페이지** (랜딩 이후 첫 화면)
   - 당뇨 / 고혈압 / 비만 / 심혈관질환 / 기타
   - 현재는 **당뇨만 활성화**, 나머지는 "준비 중" 표시
   - 당뇨를 클릭하면 → 설문 플로우로 이동

2. **당뇨 온보딩 설문 (콜드스타트 baseline)**
   - 1화면 1질문 스텝 위자드 방식
   - 클릭 전용 (텍스트 입력 0)
   - 사용자 답변에 따른 조건부 분기
   - 완료 시 결과 요약 화면

## 3. 기술 스택 결정사항

| 항목 | 결정 | 이유 |
|------|------|------|
| 프레임워크 | **Next.js (App Router)** | 프론트+API를 한 프로젝트에서 처리. 초보자 친화적 |
| 언어 | JavaScript (JSX) | TypeScript 아닌 JS로 진행 |
| 스타일링 | CSS (globals.css 또는 CSS Modules) | 초보자가 이해하기 쉬운 방식 |
| 상태관리 | React useState | 외부 라이브러리 없이 React 기본 기능 사용 |
| 라우팅 | Next.js App Router 파일 기반 | /onboarding/diabetes → 당뇨 설문 |
| DB | 추후 결정 (일단 클라이언트 state만) | 1차에서는 프론트엔드만 완성 |

## 4. UI/UX 설계 결정사항

### 설문 방식: 스텝 위자드 (1화면 1질문)
- 한 화면에 질문 하나만 보여주고 "다음" 버튼으로 전환
- `currentStep` state로 화면 제어
- 조건부 렌더링으로 그룹별 스킵 처리
- 모바일 최적화 (스크롤 불필요)

### 디자인 톤
- 헬스케어 테마 (블루/틸 계열)
- 부드러운 둥근 카드, 그라데이션 버튼
- 깔끔한 화이트 배경

### 이탈 방지 UX 전략 7가지
1. 프로그레스 바 + 스텝 카운트 (3/10)
2. 매 화면 "왜 물어보는지" 한 줄 설명
3. "모르겠어요/건너뛰기" 항상 제공
4. Phase 전환 시 중간 격려 메시지
5. 1화면 = 1질문 원칙 (예외: Screen 10만 3개 묶음)
6. 클릭 전용 — 키보드 0회
7. 이탈 시 자동 저장 + 복귀 유도

## 5. 사용자 분류 체계 (콜드스타트 3분기)

첫 질문 "당뇨와 어떤 관계인가요?"의 답변으로 3그룹 분류:

| 그룹 | 분류명 | 트리거 답변 | 실제 화면 수 | 소요시간 |
|------|--------|------------|:-----------:|:-------:|
| A | 관리형 | "당뇨 진단을 받았어요" | 12 | ~2분 30초 |
| B | 주의형 | "전당뇨(경계)라고 들었어요" | 11 | ~2분 |
| C | 예방형 | "가족력/궁금/건강관리" | 10 | ~1분 30초 |

### 분기 규칙
- Screen 6 (치료 현황): **A그룹만** 표시
- Screen 7 (검사 수치): **A+B그룹만** 표시
- 나머지 전체 공통

## 6. 설문 플로우 전체 (스크린별)

### Phase 1 — 누구인지 파악 (3스크린, 필수)

**Screen 1: 당뇨와의 관계** (분기점)
- 타입: 단일 선택 (5개 카드 버튼)
- 선택지: 당뇨 진단 → A / 전당뇨 → B / 가족력·궁금·건강관리 → C
- 이 답변이 전체 플로우의 분기를 결정

**Screen 2: 기본 프로필**
- 타입: 그리드 선택 2세트 (한 화면)
- 성별: 남성 / 여성
- 연령대: 20대 / 30대 / 40대 / 50대 / 60대+

**Screen 3: 체형 정보**
- 타입: 듀얼 슬라이더
- 키: 140~200cm (드래그)
- 몸무게: 35~150kg (드래그)
- BMI 실시간 자동 계산 표시
- → Phase 전환 메시지: "기본 프로필 완성! 이제 절반 왔어요 💪"

### Phase 2 — 핵심 위험인자 (2~4스크린, 조건부)

**Screen 4: 가족력**
- 타입: 단일 선택 (5개)
- 부모님 / 형제자매 / 둘 다 / 없어요 / 잘 모르겠어요(스킵 옵션)

**Screen 5: 동반 상태 확인**
- 타입: 복수 선택 (토글 칩)
- 고혈압·혈압약 / 고지혈증 / 고혈당 이력 / 임신성당뇨(여성만) / 해당 없음

**Screen 6: 치료 현황** ← A그룹만
- 타입: 복수 선택 (토글 칩)
- 식이요법 / 경구약 / 인슐린 / 기타 / 관리 안 함

**Screen 7: 검사 수치** ← A+B그룹만
- 타입: 그리드 선택 2세트
- HbA1c: ~5.7% / 5.7~6.4% / 6.5~7% / 7%+ / 모름(기본 선택)
- 공복혈당: ~100 / 100~125 / 126+ / 모름(기본 선택)
- → Phase 전환 메시지: "거의 다 됐어요! 마지막 생활습관만 남았어요 🏃"

### Phase 3 — 생활습관 (3스크린, 필수)

**Screen 8: 운동 빈도**
- 타입: 단일 선택 (4개, 이모지 포함)
- 거의 안 함 / 주 1~2회 / 주 3~4회 / 주 5회+

**Screen 9: 식습관**
- 타입: 복수 선택 (토글 칩)
- 밥빵면 위주 / 단 음료 / 야식 / 채소과일 매일 / 불규칙 / 해당 없음

**Screen 10: 수면 + 음주 + 흡연** (3-in-1)
- 타입: 그리드 선택 3세트 (한 화면)
- 수면: ~5h / 5~6h / 7~8h / 9h+
- 음주: 안 마심 / 가끔 / 자주 / 매일
- 흡연: 안 핌 / 과거 흡연 / 현재 흡연

### Phase 4 — 개인화 설정 (2스크린, 필수)

**Screen 11: 목표 설정**
- 타입: 복수 선택 (토글 칩)
- 위험도 / 수치 추적 / 식단 개선 / 운동 습관 / 체중 관리 / 전체

**Screen 12: AI 채팅 건강 질문 동의** ← 다나아 핵심 기능 사전 동의
- 타입: 단일 선택 (3개)
- 좋아요, 계속 받을게요 / 가끔만 / 나중에 할게요

### 완료 화면 (Screen 13)
- 프로필 요약 (분류, BMI, 관심사)
- 위험도 등급 표시
- "내 대시보드 보러가기" 버튼

## 8. 위험도 점수 산출 로직 (내부, 사용자 비노출)

한국형 + FINDRISC 하이브리드 점수 (총 0~22점):

| 항목 | 점수 | 근거 |
|------|:----:|------|
| 연령 45세+ | 0~4 | FINDRISC |
| BMI 25+/30+ | 0~3 | FINDRISC + ADA |
| 가족력 | 0~1 | ADA + Korean |
| 고혈압/혈압약 | 0~2 | FINDRISC |
| 고혈당 이력 | 0~5 | FINDRISC (최대 배점) |
| 운동 부족 | 0~2 | FINDRISC |
| 식습관 (채소 미섭취) | 0~1 | FINDRISC |
| 흡연 | 0~1 | Korean Score |
| 음주 과다 | 0~3 | Korean Score |

위험 등급:
- 0~7: 낮음 (초록)
- 8~14: 보통 (노랑)
- 15~18: 높음 (주황)
- 19~22: 매우 높음 (빨강)

## 9. 프로젝트 구조 (권장)

```
dana-a/
├── app/
│   ├── layout.js              # 전체 레이아웃 (메타데이터, 폰트)

├── next.config.mjs
└── jsconfig.json
```

## 10. 설문 엔진(StepWizard) 핵심 로직

```
state:
  currentStep = 0           // 현재 화면 번호
  answers = {}              // { relation: 'diagnosed', gender: 'male', ... }
  userGroup = null          // 'A' | 'B' | 'C' (Screen 1 답변으로 결정)

동작:
  1. questions 배열에서 currentStep번째 질문을 가져옴
  2. 해당 질문의 showFor를 확인
     - null이면 → 모든 그룹에 표시
     - ['A']이면 → userGroup이 'A'일 때만 표시, 아니면 스킵
  3. 질문 type에 따라 해당 컴포넌트 렌더링
  4. 사용자가 선택하면 answers에 저장
  5. "다음" 클릭 → currentStep + 1 (스킵 대상이면 자동으로 다음으로)
  6. 마지막 질문 이후 → CompletionScreen 표시
```

## 11. CLI 시작 명령어

```bash
# 1. 프로젝트 생성
npx create-next-app@latest dana-a
  # 옵션 선택:
  # TypeScript? → No
  # ESLint? → Yes
  # Tailwind CSS? → No (순수 CSS 사용)
  # src/ directory? → No
  # App Router? → Yes
  # import alias? → @/* (기본값)

# 2. 프로젝트 폴더 이동
cd dana-a

# 3. 개발 서버 시작
npm run dev

# 4. 브라우저에서 확인
# → http://localhost:3000
```
