# TEAM_CHANGELOG.md

## 목적

이 문서는 팀 변경 이력의 안내 문서입니다.
상세 변경 이력은 작업자별 changelog 파일에 기록합니다.

이 방식의 목적은 아래와 같습니다.

- 같은 파일 동시 수정으로 인한 Git 충돌 줄이기
- 작업자별 변경 내역 추적 쉽게 하기
- 공용 문서는 안내와 링크 중심으로 가볍게 유지하기

---

## 운영 방식

- 개인 작업 이력은 각자 파일에 기록합니다.
- 공용 기준 변경이 있어도 상세 본문은 개인 파일에 남깁니다.
- 이 파일은 changelog 위치 안내와 운영 원칙만 유지합니다.

- 이후부터는 상세 변경 사항을 이 파일에 직접 누적하지 않습니다.
