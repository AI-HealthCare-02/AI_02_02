# 노션 템플릿 실행 상태 요약

## 현재 상태
- 노션의 `Project Setup Guide` 가 전제하는 백엔드 템플릿 파일들을 현재 프로젝트 루트에 반영했다.
- `uv sync --all-groups --frozen` 실행이 완료되었고, 루트 `.venv` 가 생성되었다.
- 루트 `.env`, `envs/.local.env`, `envs/.prod.env` 를 생성했다.
- Docker 핵심 서비스 `postgres`, `redis`, `fastapi`, `nginx` 는 현재 실행 중이다.
- `http://localhost/api/docs` 에서 Swagger UI 접속을 확인했다.

## 실제로 추가된 노션 템플릿 구조
```text
프로젝트루트/
  app/                  ← FastAPI 백엔드
  ai_worker/            ← AI Worker 템플릿
  envs/                 ← 노션 가이드용 env 예시/로컬/배포 파일
  nginx/                ← nginx reverse proxy 설정
  scripts/              ← 배포/CI 스크립트
  docker-compose.yml
  docker-compose.prod.yml
  pyproject.toml
  uv.lock
  .env                  ← 실제 Docker/FastAPI가 읽는 로컬 실행 파일

  apps/
    web/                ← 기존 Next.js 프론트
```

## 중요한 구조 해석
- `app/`
  노션에서 말하는 Swagger 화면을 띄우는 FastAPI 백엔드
- `apps/web/`
  기존에 정리해둔 Next.js 프론트
- 즉 지금 프로젝트는 `FastAPI 백엔드 + Next.js 프론트` 가 함께 있는 상태다

## 왜 Swagger 화면이 떴는가
- `app/main.py` 에서 FastAPI 문서 경로를 아래처럼 정의하고 있다.

```python
docs_url="/api/docs"
redoc_url="/api/redoc"
openapi_url="/api/openapi.json"
```

- nginx 가 `localhost:80` 에서 `/api/` 요청을 FastAPI 컨테이너로 프록시한다.
- 그래서 최종 접속 주소는 `http://localhost/api/docs` 이다.

## 실제 실행 중인 컨테이너
- `postgres`
- `redis`
- `fastapi`
- `nginx`

## AI Worker 상태
- `ai-worker` 는 템플릿에 포함되어 있지만, 이미지 빌드가 무겁고 시간이 오래 걸려 이번 확인에서는 핵심 서비스만 먼저 띄웠다.
- 노션의 Swagger 목표 화면 확인에는 `ai-worker` 가 필수는 아니다.
