- 영향 범위: `backend/services/chat/*`, `backend/core/logger.py`, `backend/services/health_question.py`, `frontend/components/*`, `frontend/app/app/layout.js`, `frontend/app/app/chat/page.js`, `.env`, `envs/example.*.env`.
- 비영향 범위: public API, DB schema, migration, 인증 계약.
- 캐시 주의: 동적 상태는 cached system prompt에 넣지 않고 request-scoped layer로만 붙인다.

## 2) 데이터 안전 (중요)
- [x] 이번 변경의 SoT(Source of Truth)를 명시했다.
- [x] 상태 전이 규칙을 정의했다.
- [x] 멱등성 규칙을 정의했다.
- [x] 동시성 / race condition 위험 구간을 확인했다.
- [x] UTC / 사용자 로컬 날짜 처리 기준을 정했다.
- [x] 실패 시 복구 경로를 정리했다.

근거:
- SoT:
  - 앱 설명: `shared/danaa_product_guide.v1.json`
  - 리포트 상태: `HealthProfile`
  - 챌린지 상태: `ChallengeService.get_overview()`
  - 오늘 아직 안 적은 질문 수: `HealthQuestionService.get_daily_missing_summary()`
- 멱등성: help snapshot과 guide modal은 읽기 전용이고, live_state도 request-scoped snapshot이라 저장 부작용이 없다.
- race condition: `.env`는 프로세스 시작 시 읽히므로 적용/롤백에는 백엔드 재시작이 필요하다.
- 날짜 기준: `datetime.now(tz=config.TIMEZONE).date()`
- 복구: `CHAT_APP_CONTEXT_MODE=off` 후 백엔드 재시작.
