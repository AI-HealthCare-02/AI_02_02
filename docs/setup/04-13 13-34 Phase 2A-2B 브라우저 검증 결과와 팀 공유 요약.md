# Phase 2A-2B 브라우저 검증 결과와 팀 공유 요약

## 한 줄 결론

Phase 2A와 Phase 2B의 핵심 목표였던 **"AI 답변 아래 설문 카드 노출 → 버튼 선택 → DB 저장 → 오른쪽 요약 반영"** 흐름이 브라우저에서 실제로 검증되었습니다.

즉, 지금은 단순히 화면만 그럴듯한 상태가 아니라:

```text
사용자 질문
→ AI 답변
→ 답변 아래 설문 카드 노출
→ 카드 버튼 클릭
→ 저장 완료 문구 표시
→ 오른쪽 미답변 질문 개수 감소
→ DB에 CHAT 출처로 실제 저장
```

까지 닫힌 상태입니다.

---

## 왜 이 검증이 중요했는가

다나아의 핵심 목적은 그냥 건강 질문에 답하는 챗봇이 아니라,
**사용자가 AI 채팅을 사용하는 흐름 안에서 건강 데이터를 자연스럽게 모으는 것**입니다.

그래서 이번 검증에서 가장 중요했던 건:

1. 카드가 실제로 보이는지
2. 클릭이 실제 저장으로 이어지는지
3. 저장된 결과가 화면 요약에도 반영되는지
4. 오른쪽 패널이 더 이상 저장 주인공이 아니라 요약 역할만 하는지

였습니다.

---

## 이번에 실제로 바뀐 것

## Phase 1

백엔드 쪽에서 먼저 고친 것:

- 현재 사용자 메시지가 LLM prompt에 빠지던 버그 수정
- 오래된 10개 history를 보내던 버그 수정
  - 이제 최신 10개를 고르고, 모델에는 시간 순서대로 전달
- 저장 0건인데도 쿨다운이 올라가던 버그 수정
- `/api/v1/chat/health-answer` rate limit 추가
- 건강 질문 프롬프트를 "항상 묻기"가 아니라 "흐름이 자연스러울 때만 묻기"로 완화

쉽게 말하면:

```text
예전:
AI가 방금 사용자가 한 질문을 제대로 못 볼 수 있었음
저장도 안 됐는데 시스템은 답한 걸로 처리할 수 있었음

지금:
방금 질문을 보고 답하고
실제로 저장된 경우에만 답한 걸로 처리함
```

## Phase 2A

프론트에서 한 핵심 변경:

- SSE `done.health_questions`를 실제로 읽도록 수정
- AI 답변 바로 아래에 인라인 설문 카드 노출
- 카드에서 선택형 버튼 클릭 시 기존 `/api/v1/chat/health-answer` 호출
- 저장 완료 시 카드 안에 성공 문구 표시

쉽게 말하면:

```text
예전:
서버가 질문 데이터를 보내도 화면이 안 씀

지금:
서버가 보낸 질문을 실제 카드로 그림
```

## Phase 2B

프론트 오른쪽 패널과 localStorage 쪽 정리:

- 오른쪽 `미답변 질문` 패널을 저장 UI에서 내리고 요약 UI로 변경
- 예전 짧은 필드명 제거
  - `sleep_duration` → `sleep_duration_bucket`
  - `breakfast` → `breakfast_status`
  - `vegetable` → `vegetable_intake_level`
  - `meal_balance` → `meal_balance_level`
  - `mood` → `mood_level`
- localStorage에 남아 있던 구필드명을 canonical 필드명으로 마이그레이션
- 인라인 카드 저장 성공 시 오른쪽 요약도 같이 줄어들도록 연결

쉽게 말하면:

```text
예전:
본문 카드와 오른쪽 패널이 따로 놀았음

지금:
본문 카드가 저장 주인공
오른쪽 패널은 오늘 남은 질문 요약만 보여줌
```

## 이번에 검증한 환경

| 항목 | 실제 실행 방식 | 쉬운 뜻 |
|---|---|---|
| 백엔드 FastAPI | Docker | 서버 로직은 컨테이너 안에서 실행 |
| Postgres | Docker | DB도 컨테이너 |
| Redis | Docker | rate limit, 로그인, 일부 상태 처리에 필요 |
| 프론트 Next.js | 로컬 build + start | 화면은 내 PC에서 직접 띄움 |
| 브라우저 검증 | Playwright + 실제 페이지 | 눈으로 보는 흐름까지 확인 |

중요한 점:

- 프론트는 `NEXT_PUBLIC_AUTH_TOKEN`을 쓰기 때문에
- `.env.local` 값을 바꾼 뒤에는 **반드시 다시 build 후 start** 해야 최신 토큰이 반영됩니다.

이번에 카드가 안 보였다가 다시 보인 이유도 여기에 있었습니다.

## 브라우저에서 실제로 확인한 결과

### 1. 카드 노출

테스트 질문:

```text
오늘 운동 추천해줘
```

확인 결과:

- AI 답변이 먼저 보임
- 그 아래에 `정서+음주` 카드가 실제로 뜸
- 카드 안에 질문 2개가 보임
  - `요즘 기분은 어때? 😊`
  - `최근에 술 마신 적 있어? 🍺`

판정:

- **성공**

의미:

- 이제 설문이 텍스트 줄글이 아니라 **답변 아래 카드 UI**로 보입니다.

### 2. 카드 클릭 후 저장 반응

실제 클릭:

```text
보통
```

화면 변화:

- 카드에 `오늘 기록에 저장됐어요.` 문구 표시
- 오른쪽 패널의 `2개 남음`이 `1개 남음`으로 줄어듦

판정:

- **성공**

의미:

- 사용자가 답한 값이 화면 안에서 즉시 반영됩니다.

### 3. DB 저장 확인

DB 조회 결과:

```text
user_id = 2
log_date = 2026-04-13
mood_level = normal
mood_level_source = chat
alcohol_today = false
alcohol_today_source = chat
```

판정:

- **성공**

의미:

- 단순 화면 효과가 아니라, 실제 DB에 `chat` 출처로 저장됐습니다.

## 팀원이 이해해야 할 핵심 포인트

### 1. 지금 고친 건 "디자인"이 아니라 "수집 루프"다

이번 작업의 핵심은 예쁜 카드 추가가 아니라:

```text
질문이 실제로 뜨고
사용자가 답하고
DB에 저장되고
오른쪽 요약까지 맞아떨어지는지
```

를 맞춘 것입니다.

### 2. 오른쪽 패널은 이제 주인공이 아니다

예전에는 오른쪽 패널이 직접 저장하는 UI 성격이 강했습니다.
지금은:

- 본문 카드가 저장 주인공
- 오른쪽 패널은 요약만 보여주는 보조 UI

입니다.

### 3. health question이 안 내려오는 건 항상 버그가 아니다

이번 검증 중 user 3에서는 카드가 안 떴는데, 이유는 프론트 버그가 아니라:

- 이미 오늘 2번 질문을 받았고
- 90분 쿨다운이 걸려 있었기 때문

이었습니다.

즉:

```text
카드가 안 뜬다
= 항상 프론트 버그
```

는 아닙니다.

백엔드 engagement 상태도 함께 봐야 합니다.

### 4. dev auth token 변경 후에는 반드시 build가 필요하다

이건 팀이 꼭 알아야 합니다.

- `NEXT_PUBLIC_AUTH_TOKEN`은 프론트에서 빌드 시점 값으로 묶입니다
- 그래서 `.env.local`만 바꾸고 `start`만 다시 하면 예전 토큰으로 동작할 수 있습니다

정리:

```text
토큰 바꿈
→ npm run build
→ npm run start
```

순서가 필요합니다.

즉 지금 상태는:

- **다나아 핵심 수집 루프는 검증 완료**

## 지금 상태가 좋은지 / 나쁜지

### 좋은 점

- 본문 카드 기반 자연 수집 루프가 실제로 동작함
- 텍스트형 설문만 남던 상태에서 한 단계 크게 나아감
- DB 저장까지 확인해서 신뢰도가 높음

### 아직 아쉬운 점

- 로그인 우회는 아직 개발용 임시 구조
- 세션 복원은 아직 없음
- 고급 개인화는 아직 off 상태

### 최종 판단

**좋습니다.**

정확히 말하면:

```text
다나아의 핵심 차별점인
"대화 속 건강 데이터 수집"
이제는 브라우저와 DB 기준으로 실제 검증된 상태
```

입니다.

## 팀 공유용 짧은 문장 버전

이번 작업으로 다나아 채팅에서 **AI 답변 아래 설문 카드가 실제로 뜨고, 사용자가 버튼으로 답하면 DB에 `CHAT` 출처로 저장되는 흐름**까지 확인했습니다. 오른쪽 `미답변 질문` 패널은 더 이상 직접 저장하는 UI가 아니라 **본문 카드 기반 기록을 요약해서 보여주는 역할**로 바뀌었습니다. 즉, 예전처럼 텍스트 질문만 보이는 상태에서 벗어나 **다나아가 원래 의도했던 “대화 속 자연 수집” UX가 실제로 동작하는 단계**까지 왔습니다.
