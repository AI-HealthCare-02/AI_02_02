## B. 데이터 무결성 게이트 (중요)
- [x] 정렬 / 집계 / 상태 전이의 중복 반영 위험을 막았다.
- [x] 멱등 처리 또는 기대 이전 상태 검증이 있다.
- [x] 동시성 / race condition / stale closure 위험을 줄였다.
- [x] UTC / 로컬 날짜 기준이 일관된다.

리뷰 근거:
- `_get_prompt_history()`는 최신 10개를 고른 뒤 시간순으로 다시 정렬해 모델 입력 순서를 고정했습니다.
- `save_health_answers()`는 `saved_fields > 0`일 때만 cooldown/count를 갱신하도록 수정했습니다.
- 세션 로딩 중 입력창을 비활성화해서 이전 대화 복원과 새 메시지 전송이 겹치지 않게 했습니다.
- 현재 user 메시지 저장과 history 조회의 병렬 실행을 제거해,
  현재 질문이 history와 `message_text`에 중복으로 들어갈 가능성을 없앴습니다.
- 날짜 기준은 기존 `DailyHealthLog` 로컬 날짜 흐름을 유지합니다.
