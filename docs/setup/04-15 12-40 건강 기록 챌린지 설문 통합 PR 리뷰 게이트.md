## B. 데이터 무결성 게이트 (중요)
- [x] 정렬 / 집계 / 상태 전이의 중복 반영 위험을 막았다.
- [x] 멱등 처리 또는 기대 이전 상태 검증이 있다.
- [x] 동시성 / race condition / stale closure 위험을 줄였다.
- [x] UTC / 로컬 날짜 기준이 일관된다.

리뷰 근거:
- `POST /chat/health-answer`는 기존 one-shot 규칙을 유지하고, `PATCH /health/daily/{today}`만 same-day replace를 허용했다.
- `pending_questions`와 `missing_summary`는 오늘 날짜에만 계산되며 과거 날짜에는 붙지 않는다.
- 카드 우선순위는 수면/아침/복약 중복을 줄이는 규칙으로 재정리했다.
