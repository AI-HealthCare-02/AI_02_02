# 윈도우10 프로젝트 셋업 가이드

## 목적
- OZ 노션의 `Project Setup Guide`를 Windows 10과 현재 프로젝트 구조에 맞게 다시 정리한 문서입니다.
- 팀장 전용 단계인 저장소 생성, 팀원 초대는 제외했습니다.
- 실제 개발 코드, Python 보조 작업, 참고 자료, 보관 자료의 경계를 분리해 Claude Code와 Git 작업이 꼬이지 않도록 했습니다.

## 현재 적용한 폴더 구조
```text
마지막 웹프로젝트/
  apps/
    web/                  ← 실제 개발 코드(Next.js)
  tools/
    python/               ← Python 스크립트/AI 보조 작업 공간
  docs/
    planning/             ← 기획서, 리서치, 브리핑 문서
    prototypes/           ← HTML 데모, 레이아웃 가이드, 설계 가이드
    setup/                ← 셋업 문서
  archive/
    ai-work/              ← 이전 Claude/Codex 작업물 보관
    bundles/              ← zip 보관
    misc/                 ← 기타 자료
  .gitignore
```

## 현재 아키텍처 해석
- 프레임워크: Next.js App Router
- 현재 실행 대상: `apps/web`
- 현재 데이터 흐름: 프론트 내부 상태와 로컬 정적 데이터 중심
- 백엔드/API: 기획상 추후 `Next.js API Routes` 방향
- DB/AI 연동: 아직 미구현, 추후 확장 예정

즉 지금은 `프론트 우선 개발` 단계입니다. 그래서 Python 가상환경은 앱 내부에 섞지 않고 `tools/python` 으로 분리했습니다.

## Git 상태
- 현재 루트는 Git 저장소로 초기화되어 있습니다.
- `origin` 은 `https://github.com/AI-HealthCare-02/AI_02_02.git` 로 연결되어 있습니다.
- `archive` 와 로컬 planning 파일은 `.gitignore` 로 제외해 불필요한 보관 자료가 팀 저장소에 섞이지 않게 했습니다.

## Python / uv 기준
- 현재 PC에서 `python` 과 `py -3.11` 은 `Python 3.11.9` 를 사용할 수 있습니다.
- `uv` 는 이미 설치되어 있습니다.
- Python 작업용 가상환경은 `tools/python/.venv` 에 둡니다.
- 이 환경은 프론트 앱용이 아니라 `데이터 정리`, `스크립트`, `AI 보조 작업`, `추후 백엔드 실험` 용도입니다.

가상환경 사용 예시:

```powershell
cd C:\Users\mal03\Desktop\레퍼런스\마지막 웹프로젝트\tools\python
.\.venv\Scripts\Activate.ps1
python --version
```

## Node / Next.js 기준
- 실제 프론트 개발은 모두 `apps/web` 에서 합니다.
- 의존성 설치와 실행은 아래 기준입니다.

```powershell
cd C:\Users\mal03\Desktop\레퍼런스\마지막 웹프로젝트\apps\web
npm install
npm run dev
```

- 현재 기준으로 `npm install` 과 `npm run build` 검증은 이미 성공했습니다.

## .env 기준
- `.env` 는 `apps/web` 안에서만 관리합니다.
- 현재는 `apps/web/.env.local` 과 `apps/web/.env.example` 를 준비했습니다.
- `.env.local` 은 로컬 전용이며 Git에 올리지 않습니다.
- `.env.example` 은 팀원 공유용 예시 파일입니다.

현재 포함한 항목:
- `NEXT_PUBLIC_APP_NAME`
- `NEXT_PUBLIC_APP_ENV`
- 향후용 placeholder: `OPENAI_API_KEY`, `ANTHROPIC_API_KEY`, `SUPABASE_*`

아직 앱 코드에서 ENV를 직접 읽고 있지는 않지만, 개발 시작 전에 표준 위치를 먼저 고정한 상태입니다.

## 어디서 무슨 작업을 해야 하나
- `apps/web`
  실제 화면 개발, 컴포넌트 수정, 스타일 수정, 라우팅, Next.js API Routes 추가
- `tools/python`
  Python 스크립트, 데이터 전처리, CSV/엑셀 정리, 간단한 AI 보조 실험
- `docs/planning`
  요구사항 확인, 기능 정의, 일정/우선순위 참고
- `docs/prototypes`
  HTML 데모, 레이아웃, 질문 흐름 참고
- `archive/...`
  이전 작업 보관용, 기본적으로 수정하지 않음

## 추천 작업 루틴
1. Claude Code는 항상 프로젝트 루트에서 시작
2. 프론트 작업은 `apps/web` 기준으로 요청
3. Python 작업은 `tools/python` 기준으로 요청
4. 참고 자료는 `docs/planning`, `docs/prototypes` 에서 읽기
5. 보관 자료는 `archive` 에 두고 구현 대상에서 제외
