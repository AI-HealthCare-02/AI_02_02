- 목적: 건강 기록/챌린지/설문 카드/세션 복원 기준을 서버 기준으로 맞추고, 같은 날 오른쪽 패널 수정과 미답변 질문 보완 카드까지 일관되게 동작하게 만든다.

- 시간/날짜: `today` 기준 same-day replace는 `PATCH /health/daily/{today}` + `source=direct`에만 허용

## 2) 데이터 안전 (중요)
- [x] 이번 변경의 SoT(Source of Truth)를 명시했다.
- [x] 상태 전이 규칙을 정의했다.
- [x] 멱등성 규칙을 정의했다.
- [x] 동시성 / race condition 위험 구간을 확인했다.
- [x] UTC / 사용자 로컬 날짜 처리 기준을 정했다.
- [x] 실패 시 복구 경로를 정리했다.

근거:
- 오늘 건강 기록 SoT: `DailyHealthLog`
- 챌린지 SoT: `ChallengeService.get_overview()`
- 채팅 세션 SoT: URL `session_id`, 없으면 서버 최신 세션
- 자동 카드 저장 경로: `POST /api/v1/chat/health-answer`는 기존 one-shot 유지
- 수동 보완/오른쪽 패널 저장 경로: `PATCH /api/v1/health/daily/{today}`는 same-day replace 허용
- race 위험: 오른쪽 패널 연속 저장과 카드 저장이 겹칠 수 있으므로 응답 후 서버값으로 재동기화
- 날짜 기준: 서버 `date.today()`를 그대로 사용, 과거 날짜/미래 날짜 규칙은 기존 유지
- 실패 시 복구: 저장 실패 시 서버 마지막 성공값으로 되돌리고, additive 응답 필드가 없어도 기존 요약 UI로 fallback

- [x] DB 변경 시 마이그레이션 전략이 있다.
- [x] DB 변경 시 롤백 전략이 있다.

- [x] API 변경 시 하위 호환성 영향과 클라이언트 영향이 정리되어 있다.
- [x] 파괴적 작업은 명시적 승인 범위 안에 있다.

근거:
- DB schema 변경 없음, migration 없음
- 롤백: additive 응답 필드 사용 중단 + 프론트 fallback 로직 유지
- 호환성: `missing_summary` 유지, `pending_questions` 추가만 함
- 챌린지 overview도 `catalog` 추가만 함
- `PATCH /health/daily/{date}` 요청 body는 flat 유지
- 파괴적 작업 없음

- [x] 외부 API 한도와 실패 조건을 반영했다.
- [x] 동기 호출과 비동기 이벤트의 책임 경계를 정했다.
- [x] 재시도 / 중복 방지 / 로깅 전략을 정리했다.
- [x] 로컬 개발 환경과 운영 환경의 차이를 기록했다.

근거:

- 동기 호출: chat/health/challenge overview fetch
- 비동기 이벤트: 프론트 `danaa:challenge-overview-refresh`, `danaa:conversation-active`
- 중복 방지: `POST /chat/health-answer`의 one-shot 유지, `PATCH /health/daily/{today}`는 direct same-day replace로 분리
- 로컬/운영 차이: 챌린지 템플릿 seed가 비어 있으면 catalog/recommended가 모두 비어 보일 수 있으므로 seed 검증 필요

## 5) 테스트 계획 (중요)
- [x] 단위 테스트 계획 또는 기존 테스트 재사용 방안을 정리했다.
- [x] 통합 / 회귀 테스트 계획을 정리했다.
- [x] 실패 경로(권한 없음, 중복 요청, 외부 API 실패)를 검증한다.
- [x] 실행하지 못하는 테스트와 잔여 리스크를 명시한다.

근거:
- 백엔드 단위/통합:
  - today `pending_questions` 포함 여부
  - same-day direct replace 허용, chat one-shot 유지
  - bundle 완료 규칙과 morning sleep-first 우선순위
  - challenge overview `catalog` 포함
- 프론트 회귀:
  - 오른쪽 패널 `1 -> 2 -> 3`, `50 -> 70`
  - 미답변 질문 클릭 시 보완 카드 표시
  - 챌린지 선택 UI 복원 + join 후 overview 반영
  - report/challenge 이동 후 session 유지
- 잔여 리스크:
  - 기존 더티 워크트리와 충돌 가능성
  - 챌린지 seed가 비어 있는 로컬 환경에서는 추천/카탈로그 테스트가 제한될 수 있음
