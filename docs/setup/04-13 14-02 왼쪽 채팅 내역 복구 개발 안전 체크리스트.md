## 2) 데이터 안전 (중요)
- [x] 이번 변경의 SoT(Source of Truth)를 명시했다.
- [x] 상태 전이 규칙을 정의했다.
- [x] 멱등성 규칙을 정의했다.
- [x] 동시성 / race condition 위험 구간을 확인했다.
- [x] UTC / 사용자 로컬 날짜 처리 기준을 정했다.
- [x] 실패 시 복구 경로를 정리했다.

근거:
- SoT:
  - 세션 목록: DB `chat_sessions`
  - 세션 메시지: DB `chat_messages`
- 상태 전이:
  - 사이드바 마운트 → 세션 목록 fetch
  - 세션 클릭 → `/chat/history` fetch → messages 교체
  - 새 대화 → 현재 `sessionId`와 `messages`만 초기화
- 멱등성:
  - 같은 세션 클릭 여러 번: 같은 history를 다시 읽는 동작
- race:
  - 스트리밍 중 세션 전환은 막거나 무시
  - history 로딩 중 입력 비활성화
