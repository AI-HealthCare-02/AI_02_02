## 2) 데이터 안전 (중요)
- [x] 이번 변경의 SoT(Source of Truth)를 명시했다.
- [x] 상태 전이 규칙을 정의했다.
- [x] 멱등성 규칙을 정의했다.
- [x] 동시성 / race condition 위험 구간을 확인했다.
- [x] UTC / 사용자 로컬 날짜 처리 기준을 정했다.
- [x] 실패 시 복구 경로를 정리했다.

근거:

- SoT: `kwanju` 브랜치의 실제 코드 (`backend/*`)
- 상태 전이: 문서는 “현재 구현”과 “후속 구현”을 분리해 기록
- 멱등성: 이번 작업은 문서/운영 메모 수정으로 데이터 상태 자체를 바꾸지 않음
- 날짜 처리: 기존 코드 기준 Asia/Seoul + ISO 8601 설명 유지
- 복구: 문서 변경만 되돌리면 됨, DB 데이터 자체는 불변

- [x] DB 변경 시 마이그레이션 전략이 있다.
- [x] DB 변경 시 롤백 전략이 있다.

- [x] API 변경 시 하위 호환성 영향과 클라이언트 영향이 정리되어 있다.
- [x] 파괴적 작업은 명시적 승인 범위 안에 있다.

근거:

- 마이그레이션: `docker compose exec fastapi uv run python -m aerich upgrade`
- 롤백: migration downgrade 사용 가능, 문서에 절차 분리
- 호환성: `OnboardingStatusResponse`, `ChallengeOverviewResponse`는 additive change
- 클라이언트 영향: 프론트 계약 변경 메모에 별도 정리
