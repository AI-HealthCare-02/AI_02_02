대시보드(출력) → 필요 데이터 → 질문(입력) 역추적 프로세스

왜 대시보드가 먼저인가?
질문을 먼저 만들면 쓸데없는 질문이 생기고, 정작 필요한 데이터가 빠집니다.
STEP 1

대시보드에 무엇을 보여줄 것인가? (출력 확정)

사용자에게 보여줄 핵심 지표 3가지를 먼저 결정합니다.
확정: ① FINDRISC 위험도 + 추이   ② 생활습관 점수   ③ 체중/BMI 변화

STEP 2

그걸 보여주려면 어떤 데이터가 필요한가? (역추적)

각 지표를 실제로 계산하려면 어떤 원시 데이터가 있어야 하는지 추출합니다.
예시: FINDRISC 위험도를 보여주려면 → 운동 빈도, 식습관, 음주량, BMI 데이터 필요

STEP 3

그 데이터를 얻으려면 무엇을 물어봐야 하는가? (질문 설계)

필요한 데이터를 수집하기 위한 최소한의 질문을 설계합니다.
결과: 일일 9문항 (아침 3 + 점심 3 + 저녁 3) + 격일 2문항

STEP 4

언제, 어떻게 질문할 것인가? (타이밍/UX)

AI 채팅 응답에 자연스럽게 삽입. 시간대별 앵커링, 최소 90분 간격, 야간 침묵.
핵심: 누적된 질문은 압축, 놓친 질문은 우선순위별 복구

핵심 원칙

모든 질문은 대시보드 지표에 연결되어야 합니다.
대시보드에 반영되지 않는 질문 = 쓸데없는 질문 → 즉시 삭제.
대시보드에 데이터가 비어있는 칸 = 질문이 빠진 것 → 즉시 추가.
