## 1. 이 문서는 무엇인가

이 문서는 DANAA 서비스에서 프론트엔드와 백엔드가 어떤 형식으로 데이터를 주고받는지 쉽게 설명한 최종 API 문서입니다.

쉽게 말하면:

- 어떤 화면에서 어떤 API를 호출하는지
- request에는 어떤 값을 보내야 하는지
- response로는 어떤 값이 오는지

를 이해하기 위한 문서입니다.

## 2. 공통 규칙

- 기본 prefix: `/api/v1`
- 인증: `Authorization: Bearer {access_token}`
- 날짜 형식: `YYYY-MM-DD`
- datetime 형식: ISO 8601
- `PATCH`는 일부 필드만 보내는 수정 방식입니다.
- 신규 API 필드명은 DB 의미와 맞춘 이름을 사용합니다.

예:

- `sleep_duration_bucket`
- `vegetable_intake_level`
- `exercise_done`
- `walk_done`

## 3. 온보딩 API

## 3-1. 동의 저장

### `POST /api/v1/auth/consent`

무슨 API인가:

- 회원가입 후 동의 정보를 저장합니다.

request 예시:

```json
{
  "terms_of_service": true,
  "privacy_policy": true,
  "health_data_consent": true,
  "disclaimer_consent": true,
  "marketing_consent": false
}
```

response 예시:

```json
{
  "consented_at": "2026-04-02T10:00:00+09:00"
}
```

## 3-2. 온보딩 설문 저장

### `POST /api/v1/onboarding/survey`

무슨 API인가:

- 사용자가 처음 가입할 때 입력한 건강 정보를 저장합니다.
- 저장 후 BMI와 초기 FINDRISC 점수를 계산합니다.

중요 수정:

- request에서는 `user_group`을 직접 보내지 않습니다.
- 사용자가 보내는 원본값은 `relation`입니다.
- 서버가 `relation`을 보고 `user_group`을 계산합니다.

request 예시:

```json
{
  "relation": "prediabetes",
  "gender": "male",
  "age_range": "45_54",
  "height_cm": 175.0,
  "weight_kg": 80.0,
  "family_history": "parent_or_sibling",
  "conditions": ["hypertension"],
  "treatments": null,
  "hba1c_range": null,
  "fasting_glucose_range": null,
  "exercise_frequency": "less_than_4h",
  "diet_habits": ["irregular_meals", "frequent_snacks"],
  "sleep_duration_bucket": "between_6_7",
  "alcohol_frequency": "once_or_twice_weekly",
  "smoking_status": "non_smoker",
  "goals": ["weight_management", "blood_sugar_control"],
  "ai_consent": "agreed"
}
```

중요:

- `ai_consent` 허용값은 현재 기준으로 `agreed`, `declined` 두 가지로 고정합니다.
- `bmi`는 사용자가 보내는 값이 아니라 서버가 계산합니다.

response 예시:

```json
{
  "health_profile_id": 1,
  "user_group": "B",
  "bmi": 26.1,
  "initial_findrisc_score": 12,
  "initial_risk_level": "moderate",
  "access_token": "new-access-token",
  "message": "온보딩이 완료되었습니다."
}
```

## 3-3. 온보딩 완료 여부 확인

## 5. Health

## 5-1. 특정 날짜 기록 조회

### `GET /api/v1/health/daily/{date}`

response 구조는 `dashboard/init.daily_log`와 동일합니다.

## 5-2. 특정 날짜 기록 저장/수정

### `PATCH /api/v1/health/daily/{date}`

request 예시:

```json
{
  "source": "direct",
  "sleep_quality": "good",
  "sleep_duration_bucket": "between_7_8",
  "breakfast_status": "hearty",
  "exercise_done": true,
  "exercise_type": "walking",
  "exercise_minutes": 30,
  "water_cups": 5
}
```

response 예시:

```json
{
  "daily_log": {
    "log_date": "2026-04-02",
    "sleep_quality": "good",
    "sleep_duration_bucket": "between_7_8",
    "breakfast_status": "hearty",
    "lunch_status": null,
    "dinner_status": null,
    "vegetable_intake_level": null,
    "meal_balance_level": null,
    "sweetdrink_level": null,
    "exercise_done": true,
    "exercise_type": "walking",
    "exercise_minutes": 30,
    "walk_done": null,
    "water_cups": 5,
    "nightsnack_level": null,
    "took_medication": null,
    "mood_level": null,
    "alcohol_today": null,
    "alcohol_amount_level": null
  },
  "field_results": {
    "sleep_quality": "accepted",
    "sleep_duration_bucket": "accepted",
    "breakfast_status": "accepted",
    "exercise_done": "accepted",
    "exercise_type": "accepted",
    "exercise_minutes": "accepted",
    "water_cups": "accepted"
  },
  "challenge_update": null
}
```

검증:

- `exercise_done=false`면 `exercise_type`, `exercise_minutes` 금지
- `alcohol_today=false`면 `alcohol_amount_level` 금지
- A 그룹 외 `took_medication` 제한
- 미래 날짜 금지
- 너무 오래된 과거 날짜는 소급 입력 제한 가능

### `exercise_type` 허용값

최종 허용값:

- `walking`
- `running`
- `cycling`
- `swimming`
- `gym`
- `home_workout`
- `other`

중요:

- 기존 문서의 `walk` 예시는 더 이상 사용하지 않습니다.
- `walk_done`는 걷기 여부 boolean이고
- `exercise_type=walking`은 운동 종류 enum입니다.

## 5-3. 최근 누락 날짜 조회

### `GET /api/v1/health/daily/missing`

response 예시:

```json
{
  "missing_dates": [
    {
      "date": "2026-04-01",
      "missing_fields": ["lunch_status", "dinner_status", "exercise_done", "vegetable_intake_level"],
      "answered_fields": ["sleep_quality", "breakfast_status"],
      "completion_rate": 0.29
    }
  ],
  "max_display": 4
}
```

중요:

- `completion_rate`는 DB에 저장된 정본 컬럼이 아닙니다.
- 조회 시점에 계산해서 내려주는 응답용 값입니다.

## 5-4. 여러 날짜 한 번에 입력

### `POST /api/v1/health/daily/batch`

request 예시:

```json
{
  "entries": [
    {
      "date": "2026-04-01",
      "source": "backfill",
      "lunch_status": "simple",
      "exercise_done": true,
      "exercise_minutes": 20
    }
  ]
}
```

## 6. Measurements

## 6-1. 측정값 입력

### `POST /api/v1/health/measurements`

체중 예시:

```json
{
  "measurement_type": "weight",
  "value": 79.5,
  "unit": "kg",
  "measured_at": "2026-04-02T08:00:00+09:00"
}
```

혈압 예시:

```json
{
  "measurement_type": "blood_pressure",
  "systolic": 128,
  "diastolic": 82,
  "unit": "mmHg",
  "measured_at": "2026-04-02T08:00:00+09:00"
}
```

## 6-2. 측정값 조회

### `GET /api/v1/health/measurements`

query:

- `type`
- `limit`
- `from_date`
- `to_date`

## 10. 예전 필드명과 새 필드명

기존 문서에서 짧게 쓰던 이름과 최종 DB 기준 이름이 다를 수 있습니다.

| 예전 이름 | 최종 이름 | 설명 |
|---|---|---|
| `sleep` | `sleep_quality` | 수면 질 |
| `veggie` | `vegetable_intake_level` | 채소 섭취 수준 |
| `foodcomp` | `meal_balance_level` | 식사 구성 |
| `exercise` | `exercise_done` | 운동 여부 |
| `walk` | `walk_done` | 걷기 여부 |
| `mood` | `mood_level` | 기분/스트레스 |
| `alcohol_amount` | `alcohol_amount_level` | 음주량 |

중요:

- `sleep_hours`는 더 이상 사용하지 않습니다.
- 지금은 `sleep_duration_bucket`처럼 구간값으로 관리합니다.
